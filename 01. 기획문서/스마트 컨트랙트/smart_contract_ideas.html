누구나 만드는 스마트 컨트랙트
— 일상 계약의 재정의

월렛 서비스 확장을 위한 스마트 컨트랙트 활용 아이디어 검토
사업부 내부 공유용
아이디어 발산 단계
2026.06

스마트 컨트랙트는 "조건이 충족되면 자동으로 실행되는 디지털 계약"입니다.
핵심 가치는 단 하나입니다. 플랫폼도, 사람도 개입할 수 없는 신뢰 구조를 코드로 만드는 것.
아래 3가지 아이디어는 모두 "지금 Web2에서 실제로 발생하는 신뢰 문제"를 타겟으로 합니다.

Core Ideas — 3 Concepts
05
프리랜서 마일스톤 정산
"다 해줬는데 돈 안 준다" — 플랫폼 없이 계약을 강제하다
에스크로 프리랜서 자동 정산 글로벌 검증
배경 & 문제
국내 프리랜서 시장(디자이너·개발자·작가)에서 "작업 완료 후 대금 미지급" 분쟁은 구조적으로 반복됩니다. 크몽·숨고 같은 플랫폼이 중재하지만, 플랫폼이 없는 직거래나 소규모 계약에서는 신뢰 외에 다른 보장 수단이 없습니다. 의뢰인 입장에서도 선불 후 먹튀 위험이 동일하게 존재합니다.
해외 유사 서비스 사례
Zenland 프리랜서 대금을 스마트 컨트랙트에 예치 → 결과물 승인 시 즉시 지급. 수수료 1% (최대 $50 상한). 비수탁형 구조로 플랫폼이 자금을 보관하지 않음.
Kleros Escrow 분쟁 발생 시 익명의 크라우드소싱 배심원단이 판결. 중앙화 고객센터 없이 탈중앙 분쟁 해결 구현.
작동 방식
01 →
의뢰인이 전체 금액을 컨트랙트에 잠금
02 →
마일스톤 단계별 납품 & 확인 (1차 시안 → 30%, 최종 납품 → 70%)
03 →
각 단계 확인 즉시 해당 금액 자동 송금
04 →
분쟁 발생 시 → 중재 모듈 또는 배심원 시스템 작동
Web2 대비 차별점
기존 Web2
✕플랫폼 수수료 10~20%
✕플랫폼 정책에 따라 결과 달라짐
✕직거래 시 보장 수단 없음
스마트 컨트랙트
✓수수료 1% 이하
✓조건 충족 시 자동 실행, 개입 불가
✓플랫폼 없는 직거래에서도 신뢰 보장
리스크 / 고려사항
⚠결과물 품질 판단은 코드로 자동화 불가 → 분쟁 해결 모듈 설계가 핵심 난제
⚠국내 전자금융거래법 및 에스크로 규제 검토 필요
10
개인 적금 / 지출 컨트랙트
"나 자신과의 계약" — 의지력을 코드로 강제하다
강제 저축 지출 관리 Commitment Device 월렛 온보딩
배경 & 문제
사회초년생·20대를 중심으로 "저축하고 싶지만 지켜지지 않는" 문제는 보편적입니다. 카카오뱅크 세이프박스, 은행 아이적금 같은 상품이 있지만, 결국 "해지 요청하면 풀린다"는 한계가 있습니다. 행동경제학에서는 이를 Commitment Device(사전 구속 장치)로 정의합니다. 미래의 내가 유혹에 흔들리지 않도록, 현재의 내가 구조적으로 막아두는 방식입니다.
해외 유사 개념 & 서비스
StickK 목표 미달 시 원하지 않는 단체에 자동 기부되는 Web2 Commitment 서비스. 행동경제학 기반. 단, 자금 잠금은 불완전.
DeFi 적금 스마트 컨트랙트로 일정 기간 자금을 완전 잠금. 조건 미충족 시 인출 자체가 코드 수준에서 불가능한 구조.
작동 방식
01 →
사용자가 조건과 금액을 직접 설정 (예: 6개월 잠금, 월 30만원)
02 →
컨트랙트에 자금 잠금 — 조건 충족 전까지 인출 불가
03 →
지출 목표 연동 시 → 초과 지출 감지되면 다음 달 자동 차감 또는 벌금 적용
04 →
조건 달성 시 → 적립금 자동 반환 (보너스 옵션 가능)
Web2 대비 차별점
기존 Web2
✕언제든 해지 가능 (의지력에 의존)
✕은행 영업시간·정책 의존
✕조건 커스터마이징 불가
스마트 컨트랙트
✓코드 수준에서 인출 불가 — 진짜 강제
✓조건을 내가 직접 설계
✓월렛 서비스 첫 진입점으로 활용 가능
리스크 / 고려사항
⚠긴급 상황 시 인출 불가 → 비상 해제 조건 설계 필요 (예: 멀티 서명 동의)
⚠지출 데이터 오라클 연동 필요 (카드사 API 등) — 초기 구현 복잡도 높음
11
친구끼리 계 컨트랙트
한국 전통 '계'의 Web3 버전 — 먹튀를 코드로 막다
한국 특화 계 / 공동 저축 먹튀 방지 킬러 유즈케이스
배경 & 문제
한국의 '계'는 수십 년간 유지된 소규모 공동 금융 문화입니다. 그러나 매년 계주(방장) 먹튀 사건이 반복되고, 피해 금액도 수백만~수천만 원에 달합니다. 카카오페이 더치페이로 납입을 관리하더라도, 계주가 돈을 들고 사라지는 것을 막을 방법이 없습니다. 이 문제는 "신뢰 외에 다른 보장이 없는 구조"에서 비롯되며, 스마트 컨트랙트가 가장 직접적으로 해결할 수 있는 사례입니다.
해외 유사 서비스 사례
PoolTogether 여러 명이 자금을 풀에 예치 → 스마트 컨트랙트가 이자를 생성 → 랜덤 당첨자에게 자동 지급. 손실 없는 복권 모델로 전 세계 수백억 예치.
탈중앙 ROSCA ROSCA(순환 저축 신용조합)를 스마트 컨트랙트로 구현한 프로토콜. 아프리카·동남아 등 계 문화권에서 실제 도입 사례 증가 중.
작동 방식
01 →
계주가 컨트랙트 생성 — 인원 수·금액·순번·주기 설정
02 →
참여자 전원 납입 → 컨트랙트에 자동 잠금
03 →
설정된 순번·날짜에 따라 해당 순서의 사람에게 자동 지급
04 →
미납 발생 시 → 자동 알림 + 해당 회차 지급 보류 처리
Web2 대비 차별점
기존 Web2
✕계주가 자금 보관 → 먹튀 가능
✕분쟁 시 민사소송 외 방법 없음
✕카카오페이도 결국 계주 신뢰 의존
스마트 컨트랙트
✓자금을 컨트랙트가 보관 — 계주도 접근 불가
✓순번·지급 자동 실행, 개입 불가
✓한국 문화 특화 킬러 유즈케이스
리스크 / 고려사항
⚠중도 탈퇴 처리 로직 설계 필요 (미납 시 잔여 참여자 처리 방식)
⚠유사수신행위 규제 해당 여부 법률 검토 필요
세 아이디어 모두 "신뢰 문제를 코드로 해결"이라는 공통 축을 가집니다.
