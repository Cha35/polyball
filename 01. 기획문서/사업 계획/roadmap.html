KBO 시즌 종료(9/6)를 전환점으로, 4개 테마의 업무 순서·정의·KPI를 정리한 팀 공유 문서

01 전체 흐름 Overview
한 줄 요약
6월 수기 운영으로 테스트 즉시 개시 (개발 병렬) → 7~8월 데이터 기반 제작 (개편·SEO·광고 지면) → 9월 전환 (비야구 메인 · 기프티콘 보상) → Q4 안착 (ver 2.0 · 광고 패키지 · UGC)

9월에 KBO 시즌이 끝나면 맞힐 거리(야구 경기)와 받을 상품(직관 티켓)이 같이 사라짐. 시즌이 살아있는 지금, 둘의 대체재가 통하는지를 작게 검증하고 시즌 종료 시점에 메인을 전환하는 것이 이번 분기의 핵심임.

진행 원칙 4개

  • 원칙 1  개발 없이 시작할 수 있는 건 제일 앞 — 수기 생성·운영으로 먼저 개시하고, UI·개발은 병렬 진행
  • 원칙 2  9월 전환에 필요한 개발은 9/6에서 역산 배치
  • 원칙 3  데이터가 있어야 잘 만들 수 있는 기획(홈 개편 등)은 테스트 데이터 이후
  • 원칙 4  9월 이후로 미뤄도 되는 건 전부 뒤로 (랭킹·배지는 백로그)

GOAL
소액 기프티콘 응모 테스트로 수요 검증 후, 티켓 종료에 맞춰 메인 보상 전환DAU 정의 자체가 "매일 응모권 1매 이상 획득 유저"라서, 보상의 매력도가 DAU 목표를 직접 좌우하는 줄기
시점우선
순위		업무무슨 일인지나오는 것
6월~P0 기프티콘 응모 테스트
(테스트 B)		 폴리볼의 보상 = 야구 직관 티켓이 아닌 '소액 기프티콘'부터 시작하여 보상을 변경하는 것에 대한 테스트. 주간 응모에 기프티콘 상품(500~5,000원)을 티켓과 병행 노출 테스트 스펙, 측정 기준, 상품 소싱서버 개발 범위 확인 + 법령 검토 1회 포함

기준치 확정 절차

테스트 A·B의 go/no-go 기준치(예: 비야구 참여율 70%)는 현재 잠정값임. 테스트 개시 1주차 데이터로 베이스라인을 측정한 뒤 팀 합의로 기준치를 확정하고, 이후 변경하지 않음. 기준을 미리 못 박지 않으면 9월에 판정 자체가 논쟁이 되기 때문.
08 테마 간 연결 · 리스크

테마 간 연결 — 순서의 근거

테마 1 콘텐츠비야구 마켓이 늘어나면 광고를 붙일 지면이 늘어남
→
테마 3 광고지면에 광고를 팔아 수익 확보 (네트워크 + 직광고)
→
테마 2 보상광고 수익이 기프티콘 보상 예산이 됨 (광고주 보상 포함)
→

그래서 6~8월은 테마 1·2가 주력, 테마 3은 병행 준비, 테마 4는 최소 투입 — 이 배분이 리소스 계획의 골격임.

리스크 플래그

R-01 · 9월 DAU는 방어가 아니라 성장 목표 (8월 9,674 → 9월 11,319, +17%)
시즌 종료 월에 성장이 깔려 있음 — 비야구 전환이 "버티기"가 아니라 "성장 견인"을 전제함. 테스트 A의 go 판정 기준이 그만큼 중요하며, 미달 시 9월 목표의 현실화 논의 필요.
R-02 · 광고 단가 하락 가정 (지수 350 → 200)
매출 성장이 단가가 아닌 DAU·시청 횟수에 전적으로 의존하는 구조. 인당 시청 8회를 만들려면 광고 지면 확대(RV·DA)가 일정대로 가야 하고, 시청 경험의 마찰(이탈) 관리가 병행돼야 함.
R-03 · 보상 원가의 광고 수익 연동
기프티콘이 메인 보상이 되면 보상 원가가 광고 수익에 연동됨. 직광고 미달 시 지급 규모를 조정하는 룰을 9월 전환 전에 합의해둘 것 (광고주 보상 적용이 원가 부담을 낮추는 보완 장치).
R-04 · 법령 검토 1회
기프티콘 보상·추첨 구조는 무료 획득 원칙 유지가 전제. 테스트 B 기획 확정 시점에 법령 검토 1회 진행.
