04 화면 정의
MK-01 알림 설정 — 마케팅 토글 추가 알림함 → 우상단 "설정" → 알림 설정 페이지
Description
OVERVIEW화면 개요
기존 알림 설정 페이지(예측 알림 / 응모 알림 토글)에 마케팅 알림 토글을 신규 추가
서비스 알림(예측/응모)과 마케팅 알림을 구분선으로 분리하여 영역 구분
이 페이지는 현재 기능정의서에 미포함 상태이므로, 기존 토글 포함 전체 페이지 스펙을 정의
COMPONENTUI 구성
헤더: ← 뒤로가기 + "알림 설정" 타이틀
서비스 알림 영역:
예측 알림 토글 — "내 예측 결과와 새로운 예측 소식"
응모 알림 토글 — "신규 경기 응모 소식, 당첨 안내 등"
구분선: 굵은 구분선 (다른 섹션 구분, 디자인 가이드 §6)
마케팅 알림 영역:
이벤트·혜택 알림 토글 — "이벤트, 프로모션 등 광고성 정보 수신"
기본값: OFF
토글 라벨에 "광고성 정보" 표현 포함 필수 (정통망법 §50)
BEHAVIOR동작 정의
OFF → ON 전환: 토글 즉시 전환 + 서버 동의 상태 ON 저장 + 동의 일시·경로(MK-01) 로깅
ON → OFF 전환: 토글 즉시 전환 + 서버 동의 상태 OFF 저장 + 발송 목록 즉시 제외 + 토스트 "이벤트·혜택 알림이 해제되었어요" 노출 (정통망법 §50의5 처리 결과 통보)
앱 푸시 권한 미허용 상태에서 ON 전환 시: OS 시스템 설정으로 이동 유도 팝업 → 허용 후 복귀 시 동의 상태 ON 저장 추후 결정
MK-05(마이 → 설정)와 동기화: 한 쪽에서 ON/OFF 변경 시 다른 쪽 즉시 반영
Spec

Description
OVERVIEW화면 개요
CAL-03 "오픈 알림받기" 버튼 탭 시 노출되는 푸시 동의 모달에 마케팅 동의 체크박스를 선택 항목으로 추가
유저가 "이 경기 오픈 알림을 받겠다"는 의사를 표현한 시점이므로, "이벤트 알림도 받을래?"로 자연스럽게 확장
서비스 알림(오픈 알림)과 마케팅 알림은 별도 체크박스로 분리 (개보법 §15)
COMPONENT모달 구성
모달 타이틀: "알림을 받으시겠어요?"
체크박스 1 (필수/사전 체크): "응모 오픈 알림받기" — 서비스 알림
체크박스 2 (선택/미체크): "이벤트·프로모션 등 광고성 정보도 함께 받기" — 마케팅 동의
이미 마케팅 동의 ON 상태인 유저에게는 체크박스 2 미노출 또는 체크 상태로 노출
버튼: 취소 / 확인
BEHAVIOR동작 정의
"확인" 탭 시:
체크박스 1 체크 → ENTRY_OPEN 알림 등록 (기존 CAL-03 동작)
체크박스 2 체크 → 광고성 정보 수신 동의 ON + 동의 경로(MK-02) 로깅
체크박스 2 미체크 → 마케팅 동의 상태 변경 없음. 오픈 알림은 정상 등록
"취소" 탭 시: 모달 닫힘, 알림 등록 안 됨
앱 푸시 미허용 시: OS 시스템 설정 유도 팝업 선행 → 허용 후 본 모달 노출
Spec
