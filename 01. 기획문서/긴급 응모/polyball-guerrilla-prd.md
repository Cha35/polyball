## 01 목적 및 목표

### 목적

예비자 전원 포기로 발생하는 티켓 로스를 최소화하고, 이를 광고 기반 수익화 구좌로 전환한다.

### 목표

**1) 티켓 손실 방지**
- 당첨자/예비자 전원 포기로 확정된 잔여 티켓을 당일(월요일 17시) 이내에 100% 소진하고 실제 관람으로 전환한다.
- 처리되지 않은 티켓은 비용 손실로 기록되므로 소진율이 핵심 성과 지표다.

**2) 광고 BM 구좌 가능성 검증**
- 유저가 티켓 도전 과정에서 광고를 자발적으로 시청하는 구조를 만들어 기존 서비스 내 광고 수익화 구좌로서 유효한지 검증한다.
- 이벤트 참여율, 광고 완료율, 당첨 후 전환율을 측정해 상시 운영 가능 여부를 판단한다.

---

## 02 배경

- 정규 응모 → 예비자(2배수) 순차 인계에도 전원 포기 시 티켓 로스 발생. 실제로 이번 주 SSG 경기 2건에서 로스 발생.
- 로스 티켓은 별도 처리 없을 시 비용 손실로 기록되며, 비인기 구단 경기(SSG, NC 등)에서 반복 발생 가능성이 높음.
- 기존 광고 SDK가 배포 예정으로 붙어 있어 광고 시청 기반 이벤트를 추가 개발 없이 빠르게 구현 가능.

## 03 유저 플로우

```
1. 이벤트 진입
   응모 메인 상단 배너 탭 → 이벤트 경기 목록으로 이동
   배너는 이벤트 활성 시에만 노출

2. 경기 선택
   로스 발생 티켓 목록에서 원하는 경기 카드 탭
   → 바텀시트로 방식 안내 + 남은 도전 횟수 확인

3. 광고 시청
   "광고 보고 도전하기" 탭 시 광고 SDK 호출
   광고 시작 시점에 도전 횟수 선차감

4. 스크래치 → 결과 확인
   광고 completed 이벤트 수신 시 서버에서 추첨 결정
   → 스크래치 화면 자동 전환 → 당첨/낙첨 결과 표시

5. 당첨 처리 또는 재도전
   당첨 시 티켓 받기 / 포기 선택
   낙첨 시 동일 경기 재도전 또는 다른 경기 선택
```

## 04 정책

### 참여 대상
- 폴리볼 앱 전체 가입 유저
- 이벤트 오픈 푸시는 해당 경기 홈팀 또는 원정팀을 마이팀으로 설정한 유저에게만 발송

### 예비자 정책과의 관계
- 예비자(최초 당첨자 수의 2배수) 정책은 정규 응모 프로세스에 해당하며 긴급 티켓 이벤트와 무관하게 별도 운영
- 예비자 전원 포기/무응답 확정 이후 잔여 티켓이 긴급 티켓 이벤트로 이관되는 구조

### 이벤트 오픈 조건
- 매주 목요일, 프로모티브로부터 다음 주 당첨자 명단 확정 후 예비자 전원 포기로 잔여 티켓 발생 시 운영팀이 수동으로 이벤트 오픈
- 이벤트 오픈/종료는 운영자가 직접 제어 (자동 오픈 없음)
- 잔여 티켓 전량 소진 시 이벤트 자동 종료

### 도전 횟수
- 티켓 1장당 유저 기준 최대 5회 도전 가능
- 도전 횟수는 광고 시작 시점에 선차감
- 광고 completed 이벤트 수신 시에만 추첨 진행
- 광고 중단(closed/skipped) 또는 앱 강제 종료/네트워크 오류 시 횟수만 차감되고 추첨 없음

> **엣지케이스** — 앱 강제 종료나 네트워크 오류 시 SDK 이벤트가 미수신될 수 있음. 광고 시작 시점 선차감 방식으로 이를 방지.

### 추첨
- 추첨 결과는 광고 completed 이벤트 수신 시점에 서버에서 결정
- 스크래치 UI는 결과 연출 목적이며 추첨 로직과 무관
- 티켓별 잔여 수량 기준으로 당첨 처리 (동시 요청 시 선착 처리)

### 당첨 처리
- "티켓 받기" 선택 시 티켓 지급 프로세스 진행
- "이번엔 괜찮아요" 선택 시 해당 티켓 풀에 반환, 타 유저에게 기회 이전
- 당첨 포기한 유저는 해당 티켓에 재도전 불가 (영구 차단). 단, 다른 티켓 도전은 가능.

### 티켓 마감
- 해당 티켓 당첨자 확정 시 즉시 비활성화, 신규 도전 불가
- 잔여 티켓 0장 시 이벤트 자동 종료, 배너 미노출 처리

## 06 미결 사항

- `[운영]` 목표 배포일 미확정 — 다음 주 로스 발생 가능성 대비 최대한 빠른 배포 필요
- `[운영]` 제세공과금 대상 티켓(NC, 키움) 발생 시 처리 프로세스 별도 정의 필요
- `[개발]` 동시 다수 요청 시 당첨 처리 정합성 보장 방안 (Race condition) 확인 필요
- `[개발]` 광고 BM 구좌 유효성 판단 기준 및 측정 기간 미설정
