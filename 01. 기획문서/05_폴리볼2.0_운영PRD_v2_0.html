배경

기존 폴리볼은 야구 승부예측 → 직관티켓 응모의 단일 보상 구조로, 야구 TAM(시장 한계)에 묶여 있었다. 폴리볼 2.0은 콘텐츠 카테고리를 확장(이스포츠·연예·이슈)하고 신규 재화(볼(P))·랭킹·신규 상품 응모를 도입해, 픽/응모 빈도와 광고 노출 총량을 늘리는 것을 목표로 한다.

목표 (BM 연결)
BM 목표: 광고 매출. 픽 시(AD①)·응모 시(AD②) 광고 노출이 매출원이며, 카테고리 확장 → 픽 빈도 증가 → AD① 총량 증가, 신규 상품 응모 신설 → AD② 발생, 매일 광고·낙첨 환급 → 추가 광고 노출로 연결된다.

명칭 통일(v2.0): 구 문서의 "폴"은 전부 볼(P)로 통일한다. 화면·어드민·DB 코드(PBALL) 표기 일관 유지.
3랭킹 (MSI 우선 적용)

월드컵은 별도 랭킹 페이지가 있으므로, 신규로 상단 버튼에 랭킹을 신설한다(MSI 랭킹 먼저 적용). 개인 랭킹과 팀(친구 초대) 랭킹 2종이며, 보상은 둘 다 볼(P)로 지급하고 중복 수령을 허용한다.

팀 랭킹 (친구 초대 · 2명 고정)

친구를 초대해 2명 고정 팀을 구성한다(나 + 초대한 친구 1명). 팀 내 더 많이 적중한 사람은 '롤잘알' 뱃지(20개), 적게 적중한 사람은 '롤알못' 뱃지(5개)가 붙는다. 비기면 둘 다 5개.

4응모

응모 탭을 재화 기준 2탭으로 분리한다. 직관티켓(응모권)은 기존 유지, 신규상품(볼(P))은 신설한다. 신규상품은 회차(round)·코드풀 백본으로 운영하며(ADM-PRD v2.0), 추첨 방식은 기존 가중 추첨을 따른다(확정보상 적용분은 §5 참조).

탭상품재화
직관티켓KBO 직관티켓 (경기별) — 기존응모권
신규상품 신규키보드·마우스 등 이스포츠 타겟 경품 + 보편적 기프티콘류 (초기 리스트)볼(P)
회차·코드풀 운영 (v2.0)
운영 단위 회차(round): 응모는 회차 단위로 묶이고, 한 회차에 경품 N개를 세팅한다(회차×경품). 경품은 경품 마스터에서 재사용하고, 디지털 경품은 코드풀(바코드/PIN 재고)에서 당첨자에게 1:1 매핑 지급한다. 회차 저장 시 당첨 인원 ≤ 코드 재고 검증(실물 제외). 상세는 ADM-PRD v2.0.
회차 노출·기간·일시정지: 회차는 노출 기간 / 응모 기간을 분리해 운영한다(노출 ⊇ 응모) → 오픈 예정 티저, 응모 마감 후 상품 열람을 모두 기간으로 조절. 신규상품 탭은 회차 칩(텍스트, 예 "7월 2주"·스폰서 "올리브영 위크")으로 동시 진행 회차를 구분한다. 회차는 정규 / 스폰서(광고주 캠페인) 유형이 있으나 UI는 동일(칩 텍스트만 다름). 일시정지(off)는 노출·응모·발표를 멈추되 소진 볼(P)을 보존(재on 복귀), 크리티컬 사고 시에만 서버에 재화 재분배(환불) 별도 요청. 상태 머신은 ADM-PRD v2.0 PRD-07 / 화면은 SD-4·SD-5.
본인인증 응모 게이트 (v2.0 신규)
응모 시점부터 본인인증 필수. 당첨자 선정·경품 전달을 위해 응모자는 최초 1회 본인인증을 완료해야 하며, 미인증 시 응모할 수 없다. 인증 정보는 계정에 저장돼 재진입·수령 단계에서 재사용(스킵). 변경 핵심: 기존 수령 단계 인증 → 응모 단계 인증으로 앞당김. 응모 모집단이 전원 인증자가 되어 발표·수령 마찰 감소, 부정·중복 응모 억제. (안내 문구: "당첨자 선정 및 경품 전달을 위해 본인인증이 필요해요(최초 1회). 본인인증을 하지 않으면 응모할 수 없어요" — SD-4 GATE)
매일 광고 보고 볼(P) 받기
신규상품 응모 탭에 "광고 보고 볼(P) 받기" 신설 — 하루 시청 5회. 1~4번째 인터스티셜(전면) 각 1개, 5번째 리워드 비디오(RV) 2배(2개) → 합계 하루 6개. 5번째를 2배로 둬 끝까지 시청 유도. 재화 적립의 핵심 AD 장치(인터스티셜RV 정의서 계승).

상품 수량·당첨 인원·응모 단가(상품당 1개 고정)·교체 주기는 운영값으로 §7에 위임.

당첨 수령 프로세스

응모 완료(본인인증 충족) → 당첨자 발표 → 최다 응모자(확정)+추첨 선정 → 앱 결과 확인 → 지갑에 당첨 상품 노출 → 수령 분기: 디지털(코드풀 매핑 코드 발급, 지갑에서 바로) / 실물(배송지 입력). 제세공과금 대상(경품 가치 50,001원 이상 자동) 상품은 수령 전 안내·동의(유형 무관, 독립 축). 본인인증은 응모 시점 완료라 수령 단계엔 없음. 상세는 ADM-PRD v2.0·SD-7 참조.

5광고 · 확정보상

신규상품 응모 일부에 확정보상을 적용한다. 전체가 확정은 아니고 일부 비중만 최다 응모자에게 확정 지급하며, 나머지는 추첨한다.

확정보상 구조
예시: 어떤 경품이 5명을 뽑는다면 → 최다 응모자 1명 확정 당첨 + 나머지 4명 추첨. 확정/추첨 비중은 상품(경품)별 설정(확정 인원 N).
항목정책
확정 대상해당 경품 최다 응모자 상위 N명(확정 인원, 기본 1)
나머지당첨 인원 − 확정 인원 = 추첨 (기존 가중 추첨)
최다 응모자 표기발표 전까지 닉네임만 노출, 수량 비공개(확정 인원만큼)
동점 처리해당 수량에 먼저 도달한 사람 (수량 desc, 도달시각 asc). 예) a 30개 표기 중 b가 30개 → a 유지, b가 31개 → b로 전환
산정 방식entry_log 집계로 확정 N명 산정 → 잔여 인원 가중 추첨. winner.win_type=confirmed/draw 기록(ADM-PRD v2.0)
발표 시 공개최다 응모자의 응모 수량 공개 + 전체 당첨자 닉네임 공개
기준 고지"N명 중 1명 확정 + M명 추첨" 비중을 이벤트/응모 안내에 명시
BM 효과: "1등 하면 확정" → 볼(P)을 계속 모으고 매일 광고를 본다(AD↑). 최다 응모자 닉네임 실시간 노출 + 수량 비공개 = 추월 긴장감 → 추가 볼(P) 소비 → 광고 시청. 발표 시 수량 공개로 경쟁 서사 완성.
리스크: 2등 그룹(많이 썼는데 추월 실패)의 허탈감. → §6 낙첨 환급으로 케어. 고가 상품(주변기기)에 확정보상 적용 시 독식 논란 가능 → 초기엔 저비용 상품 위주 권장.
6낙첨 케어 (광고 환급)

당첨 결과 확인 시, 응모에 쓴 볼(P)을 모두 잃으면 이탈 위험이 크다. 낙첨자는 광고를 보면 응모한 볼(P)을 일부 환급받을 수 있다.
