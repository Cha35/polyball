04응모 연동 로직
당첨자 수 차감 공식
응모 노출 당첨자 수 = 원 당첨자 수 − 확정 티켓 사용자 수
결과값이 0 이하이면 해당 회차 응모 미오픈 처리
EXAMPLE케이스 예시
케이스 A — 원 당첨자 2명 / 확정 티켓 1명 → 응모 오픈, 당첨자 수 1명 노출
케이스 B — 원 당첨자 2명 / 확정 티켓 2명 → 응모 미오픈 처리
케이스 C — 원 당첨자 2명 / 확정 티켓 0명 → 응모 오픈, 당첨자 수 2명 (정상)
LOGIC처리 로직
확정 티켓 선택 완료 즉시 → 해당 회차 당첨자 수 실시간 차감
응모 캘린더 내 미래 경기 당첨자 수도 즉시 반영
응모 미오픈 시 → 해당 회차 응모 화면 미노출 처리 (마감 처리)
응모 오픈 알림 기능 없음 → 미오픈 알림 처리 불필요
응모 미오픈 상태 응모 캘린더 표시 방식 → 미정의 (기획 필요)
05프리미엄 마켓
CONCEPT개념 및 BM 연결 논리
광고(RV) 시청을 완료한 유저만 참여할 수 있는 마켓
적중 시 일반 마켓보다 많은 응모권 지급 — 광고 시청의 즉각적 인센티브
적중 수는 일반 마켓과 동일하게 +1 반영 — 랭킹 산정 기준 변경 없음
BM 연결: 랭킹 상위권 목표 유저 → 적중 수 극대화 위해 참여 가능 마켓 최대 소화 → 프리미엄 마켓까지 자연 진입 → 광고 시청 발생
광고 시청이 랭킹을 직접 올리는 구조가 아니라, 랭킹 경쟁 욕구가 광고 시청으로 이어지는 경로를 설계하는 것

Policy
항목내용상태
진입 조건 해당 마켓 진입 시 광고(RV) 시청 완료 후 참여 가능 확정
랭킹 반영 적중 시 일반 마켓과 동일하게 적중 수 +1 — 가중치 없음 확정
응모권 보상 일반 마켓 적중 보상보다 많이 지급 — 구체적 수치 미정의 미정의
주간 마켓 수 미정의 — 일반 마켓 대비 비중 검토 필요 미정의
일일 참여 한도 미정의 미정의
플랫폼 앱 전용 (광고 시청이 앱 전용이므로) 확정
주요 타깃 랭킹 상위권을 노리는 유저 — 적중 기회 극대화 동기로 자발적 진입 확정
NOTE일반 마켓 vs 프리미엄 마켓 비교
공통점: 적중 시 랭킹 적중 수 +1 (동일)
차이점: 프리미엄 마켓은 광고 시청 진입 비용 대신 응모권 보상이 더 많음
광고 시청이 랭킹에 직접 기여하지 않음 — 랭킹 상위권 욕구 → 더 많은 마켓 참여 → 프리미엄 마켓까지 소화 → 광고 발생의 간접 경로
프리미엄 마켓 수가 너무 적으면 경쟁 유저도 굳이 광고를 보지 않을 수 있음 — 마켓 수 설계 시 참여 동기 충분성 검토 필요
