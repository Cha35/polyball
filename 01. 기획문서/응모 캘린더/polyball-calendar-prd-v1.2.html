01개요 및 배경

폴리볼은 승부 예측 콘텐츠를 통해 응모권을 획득하고, 획득한 응모권으로 직관 경기에 응모해 티켓을 받을 수 있는 서비스다.

현재 응모는 주간 단위로 운영되며, 기존 응모 메인 페이지에서 해당 응모 기간 내 경기 목록을 확인할 수 있다. 그러나 유저가 과거 응모 이력, 현재 진행 중인 응모, 미래 예정 경기를 한눈에 파악할 수 있는 전용 뷰가 없어, 응모 참여 동기 및 서비스 재방문율을 높이는 데 한계가 있다.

이에 따라 응모 캘린더 전용 페이지를 신규 개발하여 유저의 응모 여정 전반을 시각적으로 제공하고자 한다. 기존 응모 메인은 현행 구조를 유지하며, 응모 메인 내 배너형 버튼을 통해 캘린더 전용 페이지로 진입하는 구조로 제공된다.

02목표
Goals
  • ✅유저가 응모 가능한 경기를 캘린더 형태로 직관적으로 파악할 수 있게 한다
  • ✅과거 응모 참여 이력을 캘린더에서 시각적으로 확인할 수 있게 한다 (상세 결과는 기존 응모 상세 페이지로 연결)
  • ✅미래 예정 경기에 대한 알림 설정으로 응모 참여율을 높인다
  • ✅응원 팀 필터를 통해 개인화된 응모 경험을 제공한다
  • ✅기존 응모 메인 UX를 유지하면서 캘린더 기능을 점진적으로 제공한다
Non-Goals
  • ❌기존 응모 메인 페이지(주간 응모 목록)를 대체하거나 재편하지 않는다
  • ❌응모 상세 페이지는 이번 범위에서 변경하지 않는다
  • ❌승부 예측 콘텐츠 자체의 UI/UX는 이번 범위에 포함하지 않는다
  • ❌KBO 외 타 리그/종목은 이번 범위에 포함하지 않는다 (향후 확장 가능성 열어둠)

04사용자 시나리오
Scenario A
응모 가능한 경기를 찾아 응모한다응모 메인의 '전체 응모 일정 보기' 버튼을 탭해 캘린더로 진입. 현재 응모 가능한 경기를 확인하고, 하단 목록에서 경기를 탭해 기존 응모 상세 페이지로 이동해 응모한다.
Scenario B
내가 응모한 경기의 결과를 확인한다캘린더에서 과거 날짜를 탐색. 참여 아이콘(🎉/✓)이 표시된 경기를 탭해 기존 응모 상세 페이지로 이동하여 당첨 여부를 확인한다.
Scenario C
응원 팀의 다음 응모 경기를 미리 파악한다응원 팀 필터를 적용해 특정 구단의 경기만 필터링하고, 미래 예정 경기에 알림받기를 설정한다.
Scenario D
월간 일정을 한눈에 파악한다월간 뷰로 전환하여 이번 달 전체 경기 일정과 응모 상태를 파악하고, 특정 날짜를 탭해 하단 경기 목록을 확인한다.
05기능 요구사항
5-1 응모 메인 — 캘린더 진입 버튼
📌 기존 응모 메인 페이지는 현행 구조 유지. 캘린더는 별도 페이지로 제공되며, 응모 메인 내 배너형 버튼을 통해 진입한다.
항목내용
진입 버튼 위치응모 메인 페이지 상단 (응모 목록 위)
버튼 형태배너형 버튼 — '📅 전체 응모 일정 보기 ›'
진입 시 동작캘린더 전용 페이지로 이동
5-2 캘린더 뷰
기능설명
주간 뷰기본값. 해당 주의 경기를 날짜별로 표시. 날짜 셀에 팀명 + 상태 뱃지만 노출 (심플)
월간 뷰월 단위 그리드. 날짜 셀에 팀명 + 상태 뱃지 노출. 날짜 탭 시 하단 경기 목록 업데이트
뷰 전환주간 ↔ 월간 탭으로 전환 가능
날짜 이동이전/다음 주(월) 스와이프 또는 화살표
날짜 피커헤더 연/월 탭 시 날짜 피커로 빠른 이동
오늘로 이동현재 날짜로 즉시 이동 버튼
스크롤 위치 복원경기 상세 진입 후 복귀 시 이전 스크롤 위치 복원
5-3 날짜 셀 — 경기 상태 표시

날짜 셀 내 경기는 팀명 텍스트 + 상태 뱃지로 심플하게 표현한다. 최초 진입 시 범례를 노출하고 확인 후 자동 숨김 처리한다.

상태정의표현 방식
응모중현재 응모 기간, 응모 열림팀명 + 응모중 뱃지 (컬러 강조)
마감응모 기간 종료, 발표 전팀명 + 마감 뱃지 (흐리게)
결과당첨 결과 확인 가능팀명 + 결과 뱃지 (흐리게)
예정응모 미오픈 미래 경기팀명 + 예정 뱃지 (흐리게)

참여 아이콘 (셀 내 추가 표시)

참여 상태아이콘
당첨🎉
미당첨 (응모함)✓
미응모없음
⚠️ 응모 불가 구단(두산/한화/삼성) 홈경기는 캘린더에 미노출. 기존 응모 메인과 동일하게 해당 구단 카드 자체를 제공하지 않는다.
5-4 캘린더 하단 — 경기 목록
📌 캘린더에서 날짜를 탭하면 해당 일자의 경기 목록이 캘린더 하단에 바로 노출된다. 별도 페이지 이동 없음.
항목내용
카드 구성기존 응모 메인 카드 컴포넌트 재사용 + 응모 상태 뱃지 추가
노출 정보홈팀 vs 원정팀 / 날짜·시간·구장 / 응모 상태 뱃지 / 총 응모 수량 / 액션 버튼
액션 버튼상태에 따라 분기 (아래 5-5 참고)
기본 노출 날짜캘린더 진입 시 오늘 날짜 기준으로 자동 노출
5-5 경기 탭 시 액션 (상태별 분기)
경기 상태탭 시 동작
응모중기존 응모 상세 페이지로 이동 (현행 유지)
마감 / 결과기존 응모 상세 페이지로 이동 — 나의 참여 여부 및 당첨 결과 확인
예정응모 예정 상세 화면으로 이동 — 오픈 예정 안내 + 알림받기 CTA
✅ 응모중/마감/결과 상태의 상세 페이지는 기존 응모 상세 페이지를 그대로 활용. 이번 범위에서 별도 화면 개발 없음.
5-6 응모 예정 상세 화면

응모 예정 경기를 탭했을 때 노출되는 화면. 기존 응모 상세와 동일한 경기 정보를 제공하되, 응모 진행 영역을 오픈 예정 안내로 대체한다.

영역내용
경기 정보홈팀 vs 원정팀 / 날짜·시간·구장 / 당첨 좌석·인원·총 응모 수량 (기존 상세와 동일)
응모권 스탭퍼 영역"곧 열립니다 / 응모 오픈 예정: OO월 OO일(요일) 오전 10시" 로 대체
하단 CTA"응모하기" 대신 "🔔 응모 오픈 알림받기" 버튼

07이번 범위 외 항목

아래 항목들은 이번 버전에서 신규 개발하지 않으며, 기존 화면을 그대로 활용하거나 향후 과제로 분류한다.

항목처리 방향
응모 상세 (응모중)기존 응모 상세 페이지 그대로 활용. 이번 범위에서 변경 없음.
결과 확인 화면기존 응모 상세 페이지로 연결. 캘린더 셀에 참여 아이콘(🎉/✓)만 표시.
응모 불가 구단 처리캘린더 미노출 확정. 별도 안내 화면 없음.
응모권 미보유 유도 화면이번 범위 외. 향후 온보딩/리텐션 기획 시 검토.
비로그인 뷰현재 접근 불가 유지. SEO 고려한 확장은 별도 기획 필요.
캘린더 Option 1 (전면 배치)로드맵 등록. 지표 기반 전환 여부 추후 검토.
