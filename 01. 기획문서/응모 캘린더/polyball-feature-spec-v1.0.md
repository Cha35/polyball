## 3. 기능 상세 정의

---

### F-01. 캘린더 진입 버튼

**화면:** 응모 메인
**설명:** 응모 메인 페이지에서 캘린더 전용 페이지로 진입하는 배너형 버튼

#### 3.1.1 UI 스펙

| 항목 | 내용 |
|---|---|
| 위치 | 응모 메인 상단 — 기존 당첨 결과 배너 하단, 응모 목록 상단 |
| 형태 | 가로 전체 너비 배너형 버튼 |
| 텍스트 | 📅 전체 응모 일정 보기 › |
| 상태 | 항상 노출 (조건 없음) |

#### 3.1.2 동작

| 트리거 | 동작 |
|---|---|
| 버튼 탭 | 응모 캘린더 페이지로 이동 (Push Navigation) |

#### 3.1.3 KPI 연결

- 이벤트 로깅: `calendar_banner_click`
- 퍼널 1단계 — 캘린더 진입률 측정 기준

---

### F-02. 주간 캘린더 뷰

**화면:** 응모 캘린더
**설명:** 캘린더 진입 시 기본으로 노출되는 주간 단위 뷰

#### 3.2.1 UI 스펙

| 항목 | 내용 |
|---|---|
| 기본 뷰 | 주간 (캘린더 진입 시 default) |
| 표시 단위 | 화~월 (응모 대상 경기 기준 요일) |
| 날짜 표시 | 요일 + 날짜 숫자 |
| 오늘 날짜 | 원형 강조 표시 |
| 이동 방식 | 좌우 스와이프 또는 화살표 버튼 |

#### 3.2.2 동작

| 트리거 | 동작 |
|---|---|
| 좌 스와이프 / 다음 버튼 | 다음 주로 이동 |
| 우 스와이프 / 이전 버튼 | 이전 주로 이동 |
| 날짜 탭 | 해당 일자 하단 경기 목록 업데이트 (F-06) |
| 월간 탭 | 월간 뷰로 전환 (F-03) |

#### 3.2.3 데이터 조건

| 항목 | 내용 |
|---|---|
| 표시 범위 | 2026 KBO 시즌 전체 |
| 노출 경기 | 응모 가능 7개 구단 홈경기만 (LG, KT, SSG, NC, KIA, 롯데, 키움) |
| 미노출 경기 | 두산, 한화, 삼성 홈경기 — 셀에 표시하지 않음 |
| 경기 없는 날 | 셀 비워둠 (별도 표시 없음) |

### F-03. 월간 캘린더 뷰

**화면:** 응모 캘린더
**설명:** 월 단위 그리드 형태의 캘린더 뷰

#### 3.3.1 UI 스펙

| 항목 | 내용 |
|---|---|
| 표시 단위 | 월 단위 7열 그리드 |
| 날짜 셀 | 날짜 숫자 + 팀명 + 상태 뱃지 (F-04) |
| 이동 방식 | 좌우 스와이프 또는 화살표 버튼 |

#### 3.3.2 동작

| 트리거 | 동작 |
|---|---|
| 날짜 탭 | 해당 일자 하단 경기 목록 업데이트 (F-06) |
| 이전/다음 버튼 | 전월/다음달로 이동 |
| 주간 탭 | 주간 뷰로 전환 (F-02) |

### F-04. 날짜 셀 — 경기 상태 표시

**화면:** 응모 캘린더 (주간/월간 공통)
**설명:** 날짜 셀 내 경기 정보 및 응모 상태를 시각적으로 표현

#### 3.4.1 상태 정의

| 상태 | 조건 | 표현 방식 |
|---|---|---|
| 응모중 | 현재 응모 기간 내 해당 경기 응모 열림 | 팀명 + [응모중] 뱃지 (컬러 강조) |
| 마감 | 응모 기간 종료, 당첨 발표 전 | 팀명 + [마감] 뱃지 (흐리게) |
| 결과 | 당첨 결과 확인 가능 | 팀명 + [결과] 뱃지 (흐리게) |
| 예정 | 응모 미오픈 미래 경기 | 팀명 + [예정] 뱃지 (흐리게) |

#### 3.4.2 표시 규칙

| 항목 | 내용 |
|---|---|
| 하루 경기 수 | 최대 N개 (경기 수만큼 표시, 디자인 시안에서 최대값 확정) |
| 상태 우선순위 | 응모중 > 예정 > 마감 > 결과 순으로 강조 |
| 최초 진입 시 | 범례 팝업 노출 후 자동 숨김 (F-13) |

---

### F-05. 날짜 셀 — 참여 아이콘

**화면:** 응모 캘린더 (주간/월간 공통)
**설명:** 유저가 응모한 경기에 참여 여부를 셀 내 아이콘으로 표시

#### 3.5.1 아이콘 정의

| 참여 상태 | 아이콘 | 노출 조건 |
|---|---|---|
| 당첨 | 🎉 | 해당 경기 응모 완료 + 당첨 결과 확정 |
| 미당첨 (응모함) | ✓ | 해당 경기 응모 완료 + 미당첨 결과 확정 |
| 응모함 (결과 전) | ✓ (흐리게) | 해당 경기 응모 완료 + 결과 미발표 |
| 미응모 | 없음 | — |

#### 3.5.2 데이터 조건

- 로그인 유저의 응모 이력 API 호출 (캘린더 진입 시 1회)
- 응모 완료 직후 해당 셀 아이콘 즉시 업데이트

---

### F-06. 하단 경기 목록

**화면:** 응모 캘린더
**설명:** 날짜 탭 시 해당 일자의 응모 가능 경기 목록을 캘린더 하단에 노출

#### 3.6.1 UI 스펙

| 항목 | 내용 |
|---|---|
| 위치 | 캘린더 하단 고정 영역 |
| 기본 노출 | 캘린더 진입 시 오늘 날짜 기준 목록 자동 노출 |
| 카드 컴포넌트 | 기존 응모 메인 카드 재사용 + 응모 상태 뱃지 추가 |

#### 3.6.2 카드 노출 정보

| 정보 | 내용 |
|---|---|
| 경기 정보 | 홈팀 vs 원정팀 |
| 일시 | 날짜 · 시간 · 구장명 |
| 응모 상태 | 상태 뱃지 (응모중 / 마감 / 결과 / 예정) |
| 총 응모 수량 | 현재까지 응모된 총 수량 |
| 내 사용 수량 | 내가 해당 경기에 사용한 응모권 수 |
| 액션 버튼 | 상태별 분기 (F-10 참조) |

#### 3.6.3 동작

| 트리거 | 동작 |
|---|---|
| 날짜 탭 | 해당 일자 경기 목록으로 업데이트 |
| 경기 없는 날 탭 | "이 날은 응모 가능한 경기가 없어요" 빈 상태 표시 |
| 카드 탭 | 상태별 분기 화면으로 이동 (F-10) |

#### 3.6.4 API 조건

| 항목 | 내용 |
|---|---|
| 요청 파라미터 | 날짜 (YYYY-MM-DD) |
| 응답 데이터 | 해당 일자 응모 가능 구단 경기 목록 |
| 기존 API 재사용 | 응모 목록 API에 날짜 파라미터 추가 |

### F-07. 응원팀 필터

**화면:** 응모 캘린더
**설명:** 특정 구단의 경기만 필터링하여 캘린더 및 하단 목록에 표시

#### 3.7.1 UI 스펙

| 항목 | 내용 |
|---|---|
| 위치 | 캘린더 상단 (뷰 전환 탭 하단) |
| 형태 | 칩(Chip) 형태 필터 버튼 |
| 선택지 | 전체 + 응모 가능 7개 구단 |
| 기본값 | 전체 |

#### 3.7.2 동작

| 트리거 | 동작 |
|---|---|
| 구단 칩 탭 | 해당 구단 경기만 캘린더 셀 및 하단 목록에 표시 |
| 전체 탭 | 필터 초기화, 전체 경기 표시 |
| 필터 적용 중 날짜 이동 | 필터 상태 유지하며 주/월 이동 |

#### 3.7.3 예외 처리

| 상황 | 처리 |
|---|---|
| 선택 구단의 해당 주/월 경기 없음 | "이 기간에는 [구단명] 경기가 없어요" 빈 상태 표시 |
| 응모 불가 구단 선택 시 | 해당 구단은 필터 선택지에 노출하지 않음 |

### F-10. 경기 탭 — 상태별 분기

**화면:** 응모 캘린더 (하단 경기 목록 카드)
**설명:** 경기 카드 탭 시 경기 상태에 따라 다른 화면으로 분기

#### 3.10.1 분기 정의

| 경기 상태 | 탭 시 이동 화면 | 비고 |
|---|---|---|
| 응모중 | 기존 응모 상세 페이지 | 현행 유지, 변경 없음 |
| 마감 | 기존 응모 상세 페이지 | 결과 미발표 상태로 노출 |
| 결과 | 기존 응모 상세 페이지 | 당첨/미당첨 결과 노출 |
| 예정 | 응모 예정 상세 화면 (F-11) | 신규 개발 |

#### 3.10.2 이벤트 로깅

| 이벤트 | 파라미터 |
|---|---|
| `calendar_game_tap` | game_id, game_status, date |

### F-11. 응모 예정 상세 화면

**화면:** 응모 예정 상세
**설명:** 응모가 아직 열리지 않은 경기의 상세 정보 및 알림받기 CTA 제공

#### 3.11.1 UI 스펙

| 영역 | 내용 | 비고 |
|---|---|---|
| 경기 정보 | 홈팀 vs 원정팀 / 날짜·시간·구장 | 기존 상세와 동일 |
| 당첨 정보 | 당첨 좌석 / 당첨 인원 / 총 응모 수량 | 기존 상세와 동일 |
| 오픈 예정 안내 | "응모가 곧 열려요! / 응모 오픈 예정: MM월 DD일(요일) 오전 10시" | 스탭퍼 영역 대체 |
| 유의사항 | 기존 상세 유의사항 목록 | 기존과 동일 |
| 하단 CTA | "🔔 응모 오픈 알림받기" | 응모하기 버튼 대체 |

#### 3.11.2 동작

| 트리거 | 동작 |
|---|---|
| 알림받기 버튼 탭 | F-12 푸시 허용 분기 처리 |
| 뒤로가기 | 캘린더로 복귀 + 스크롤 위치 복원 (F-09) |

#### 3.11.3 데이터 조건

| 항목 | 내용 |
|---|---|
| 응모 오픈 예정일 | 응모 기간 기준 역산 — 해당 경기일 -2주 해당 월요일 |
| 오픈 시각 | 오전 10시 고정 |

### F-12. 알림받기 — 푸시 허용 유도

**화면:** 응모 예정 상세
**설명:** 알림받기 버튼 탭 시 앱 푸시 허용 여부를 확인하고, 미허용 시 허용을 유도

#### 3.12.1 분기 처리

| 상태 | 처리 |
|---|---|
| 푸시 허용 | 알림 구독 API 호출 → "알림이 설정되었어요" 토스트 메시지 |
| 푸시 미허용 | OS 시스템 푸시 허용 팝업 노출 → 허용 시 알림 구독 → 토스트 |
| 시스템 팝업 거부 | 토스트: "설정 > 알림에서 허용할 수 있어요" 안내 |

#### 3.12.2 알림 발송 조건

| 알림 유형 | 발송 시점 | 대상 |
|---|---|---|
| 응모 오픈 알림 | 응모 오픈 당일 오전 10시 | 해당 경기 알림받기 설정 유저 |
| 응모 마감 임박 알림 | 응모 마감 하루 전 | 해당 경기 응모 완료 유저 |
| 당첨 결과 알림 | 결과 확정 즉시 | 해당 경기 응모 완료 유저 |

### F-13. 최초 진입 범례

**화면:** 응모 캘린더
**설명:** 캘린더 최초 진입 시 상태 뱃지 의미를 안내하는 범례 팝업

#### 3.13.1 동작

| 트리거 | 동작 |
|---|---|
| 캘린더 최초 진입 | 범례 팝업 자동 노출 |
| 확인 버튼 탭 | 팝업 닫힘 + 다시 노출하지 않음 (로컬 플래그 저장) |
| 2회 이상 진입 | 범례 미노출 |

#### 3.13.2 범례 항목

## 4. 공통 조건

### 4.1 접근 권한

| 상태 | 처리 |
|---|---|
| 로그인 유저 | 전체 기능 이용 가능 |
| 비로그인 유저 | 캘린더 페이지 접근 불가 — 로그인 화면으로 리디렉션 |

### 4.2 에러 처리

| 상황 | 처리 |
|---|---|
| 경기 데이터 로딩 실패 | "데이터를 불러오지 못했어요. 다시 시도해 주세요" + 재시도 버튼 |
| 알림 구독 API 실패 | "알림 설정에 실패했어요. 다시 시도해 주세요" 토스트 |
| 네트워크 없음 | 오프라인 안내 토스트 |

### 4.3 성능 조건

| 항목 | 기준 |
|---|---|
| 캘린더 초기 로딩 | 2초 이내 |
| 날짜 탭 후 목록 업데이트 | 1초 이내 |
| 뷰 전환 (주간 ↔ 월간) | 즉시 (애니메이션 포함 300ms 이내) |
