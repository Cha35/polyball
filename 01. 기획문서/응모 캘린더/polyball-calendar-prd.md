# PRD — 폴리볼 응모 캘린더

> **문서 버전:** v1.1
> **최종 업데이트:** 2026.04.08
> **작성자:** PM
> **상태:** 유저 테스트 및 내부 검토 반영 완료

---

## 1. 개요 및 배경

**폴리볼**은 승부 예측 콘텐츠를 통해 응모권을 획득하고, 획득한 응모권으로 직관 경기에 응모해 티켓을 받을 수 있는 서비스다.

현재 응모는 주간 단위로 운영되며, 기존 응모 페이지에서 해당 주의 응모 가능 경기를 확인할 수 있다. 그러나 유저가 **과거 응모 이력, 현재 진행 중인 응모, 미래 예정 경기**를 한눈에 파악할 수 있는 전용 뷰가 없어, 응모 참여 동기 및 서비스 재방문율을 높이는 데 한계가 있다.

이에 따라 **응모 캘린더 전용 페이지**를 신규 개발하여 유저의 응모 여정 전반을 시각적으로 제공하고자 한다.

---

## 2. 목표

### ✅ Goals
- 유저가 응모 가능한 경기를 캘린더 형태로 직관적으로 파악할 수 있게 한다
- 과거 응모 참여 이력 및 당첨 결과를 캘린더에서 바로 확인할 수 있게 한다
- 미래 예정 경기에 대한 알림 설정으로 응모 참여율을 높인다
- 응원 팀 필터를 통해 개인화된 응모 경험을 제공한다
- 응모권 개념을 캘린더 내에서 자연스럽게 학습할 수 있는 경험을 제공한다

### ❌ Non-Goals
- 기존 응모 페이지(주간 응모 진행)를 대체하지 않는다
- 승부 예측 콘텐츠 자체의 UI/UX는 이번 범위에 포함하지 않는다
- KBO 외 타 리그/종목은 이번 범위에 포함하지 않는다 (향후 확장 가능성 열어둠)

## 3. 응모 구조 이해

캘린더를 설계하기 위해 기반이 되는 응모 운영 구조는 아래와 같다.

| 항목 | 내용 |
|---|---|
| 응모 기간 | 매주 월요일 ~ 일요일 |
| 응모 대상 경기 | 응모 기간 기준 +2주 후 화요일 ~ 일요일 경기 |
| 응모 가능 구단 | LG, KT, SSG, NC, KIA, 롯데, 키움 홈경기 (7개 구단) |
| 응모 불가 구단 | 두산, 한화, 삼성 홈경기 (티켓 미보유, 준비 중) |
| 시즌 범위 | 2026 KBO 시즌 |

## 4. 사용자 시나리오

### Scenario A — 응모 가능한 경기를 찾아 응모한다
> 유저가 캘린더를 열어 현재 응모 가능한 경기를 확인하고, 원하는 경기를 탭해 응모 페이지로 진입한다.

### Scenario B — 내가 응모한 경기의 결과를 확인한다
> 유저가 캘린더에서 과거 날짜를 탐색하며 자신이 응모했던 경기를 찾고, 탭하여 당첨 여부를 확인한다.

### Scenario C — 응원 팀의 다음 응모 경기를 미리 파악한다
> 유저가 응원 팀 필터를 적용해 특정 구단의 경기만 필터링하고, 미래 예정 경기에 알림받기를 설정한다.

### Scenario D — 월간 일정을 한눈에 파악한다
> 유저가 월간 뷰로 전환하여 이번 달 전체 경기 일정과 응모 상태를 파악한다.

### Scenario E — 응모 불가 구단 팬이 캘린더를 탐색한다
> 응모 불가 구단(두산/한화/삼성)의 팬이 응원팀 필터를 적용했을 때, 빈 화면 대신 '준비 중 안내 + 알림받기' 화면을 만나고 이탈 없이 알림을 설정한다.

## 5. 기능 요구사항

### 5-1. 응모권 현황 바 (캘린더 상단 고정)

유저가 캘린더를 탐색하는 내내 응모권 보유 현황을 인지할 수 있도록 상단에 고정 노출한다.

| 유저 상태 | 노출 내용 |
|---|---|
| 응모권 보유 | 잔여 응모권 수량 표시 |
| 응모권 미보유 | '승부 예측하고 응모권 받기' CTA 버튼으로 전환 |

- 응모권 수량은 캘린더 진입 시 + 응모 완료 직후 갱신
- 미보유 상태의 CTA 탭 시 승부 예측 페이지로 이동

### 5-2. 캘린더 뷰

| 기능 | 설명 |
|---|---|
| 주간 뷰 | 기본값. 해당 주의 경기를 날짜별로 표시 |
| 월간 뷰 | 월 단위 그리드. 경기 수를 닷(dot) 또는 숫자로 요약 표시. 날짜 탭 시 당일 경기 리스트 시트로 표시 |
| 뷰 전환 | 주간 ↔ 월간 탭으로 전환 가능 |
| 날짜 이동 | 이전/다음 주(월) 스와이프 또는 화살표 |
| 날짜 피커 | 헤더의 연/월 탭 시 날짜 피커로 빠른 이동 가능 |
| 오늘로 이동 | 현재 날짜로 빠르게 이동하는 버튼 |
| 스크롤 위치 복원 | 경기 상세 진입 후 복귀 시 이전 스크롤 위치 복원 |

### 5-3. 경기 상태 표시

경기 상태는 텍스트 레이블 뱃지로 명시적으로 표현하며, 최초 진입 시 범례를 노출한다. 범례는 확인 후 자동 숨김 처리한다.

| 상태 | 정의 | 표현 방식 |
|---|---|---|
| 응모 가능 | 현재 응모 기간 내, 응모 열려 있음 | 컬러 강조 + '응모중' 뱃지 |
| 응모 마감 | 응모 기간 종료, 당첨 발표 전 | 흐리게 + '마감' 뱃지 |
| 결과 발표 | 당첨 결과 확인 가능 | 흐리게 + '결과' 뱃지 |
| 응모 예정 | 아직 응모가 열리지 않은 경기 | 흐리게 + '예정' 뱃지 |
| 준비 중 | 응모 불가 구단 경기 | 회색 처리 + '준비중' 뱃지 |

**응모 참여 아이콘 (셀 내 표시)**

| 참여 상태 | 아이콘 |
|---|---|
| 당첨 | 🎉 |
| 미당첨 | ✓ |
| 미응모 | 없음 |

### 5-4. 경기 탭 시 액션 (상태별 분기)

| 경기 상태 | 탭 시 동작 |
|---|---|
| 응모 가능 | 응모 상세 페이지로 이동, 응모 진행 가능 |
| 응모 마감 / 결과 발표 | 나의 참여 여부 및 당첨 결과 확인 화면 |
| 응모 예정 | 알림받기 버튼 노출, 알림 설정 가능 |
| 준비 중 | '이 구단은 현재 응모를 준비 중입니다' 안내 + 알림받기 버튼 |

### 5-5. 필터

| 필터 | 설명 |
|---|---|
| 응원 팀 필터 | 유저가 설정한 응원 팀의 경기만 표시 |
| 전체 보기 | 필터 초기화, 전체 경기 표시 |

**응모 불가 구단 필터 적용 시 빈 상태 처리**
- '현재 OO 구단의 응모는 준비 중이에요' 안내 메시지 노출
- '응모 오픈 알림받기' 버튼 제공
- 빈 화면(empty state) 그대로 노출 금지

### 5-6. 알림

| 알림 유형 | 대상 | 발송 시점 |
|---|---|---|
| 응모 오픈 알림 | 알림받기 설정 유저 | 응모 오픈 당일 오전 |
| 응모 마감 임박 알림 | 해당 경기 응모 유저 | 마감 하루 전 |
| 당첨 결과 알림 | 해당 경기 응모 유저 | 결과 확정 즉시 |

- 알림받기 버튼 탭 시 앱 푸시 허용 여부 확인
- 미허용 상태면 푸시 허용 유도 시스템 팝업 자연스럽게 노출
- 알림 발송 시점 상세 명세는 별도 알림 기획 문서로 분리

### 5-7. 접근 권한

| 상태 | 접근 범위 |
|---|---|
| 로그인 유저 | 캘린더 전체 기능 이용 가능 |
| 비로그인 유저 | 현재 접근 불가. 향후 SEO 고려한 비로그인 뷰 확장 가능성 있음 (별도 기획 필요) |
