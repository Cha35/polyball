01
목적 및 목표
배경 및 문제 정의

현재 폴리볼의 핵심 퍼널은 픽 참여 → 응모권 획득 → 경기 응모 → 당첨으로 구성된다. 이 구조에서 경기 응모는 주 1회(월요일 발표) 단위로 운영되어, 유저가 재방문해야 할 일간 유인이 약하다. 픽 참여 자체도 "응모권 획득"이라는 도구적 동기에 머물러, 경기 결과에 대한 몰입도와 재방문 빈도가 낮다.

포인트 배팅 시스템은 이 문제를 해결하기 위해 설계된다. 유저가 경기마다 포인트를 걸고 결과에 능동적으로 개입하도록 하여, 픽 참여의 목적 자체를 강화한다. 쌓인 포인트는 기프티콘 경품 응모에 활용되며, 이는 단기 보상 루프를 형성하여 일간 재방문 빈도를 높인다.

✅ SOOP 라이브 젬 모델 준용 — 게임물 미분류 선례
포인트는 현금 구매·충전·환전 불가. 서비스 내 활동 보상으로만 획득.
포인트 → 재화 교환은 응모(확률) 형태로만 허용. 확정 교환 불가.
유료 재화(응모권)와 계통 분리. 포인트 ↔ 응모권 교환 불가.
포인트 계정 간 이전·양도 금지. 지정 경품 응모에만 사용 가능.
오퍼월(CPI/CPA 기반 광고주 과제) 제외. 플랫폼 내부 행동만 획득 트리거.
사행성 판단 기준: 한국 법령상 사행행위는 대가성 × 우연성의 결합으로 판단된다. 포인트가 무료 획득인 이상 대가성이 없으며, 응모(우연성)와 결합되어도 위법 소지가 없다. SOOP은 동일 구조로 규제 당국의 게임물 분류를 받지 않고 운영 중이다.

동적 배당 계산 방식 (SOOP 준용)
배당 공식
// 배팅 풀 구성
총 배팅 풀 (P) = A선택 총합 (Pa) + B선택 총합 (Pb)
// 배당 계산 (적중 시)
내 수령 포인트 = 내 배팅 × ( P ÷ 내가 선택한 팀 총합 )
// 예시: P=1,000 / A:700 / B:300, 내가 B에 100 배팅, B 적중
수령 = 100 × (1,000 ÷ 300) = 333P // 순수익 +233P
배당률 특성: 다수가 선택한 팀(인기 팀) 배당률이 낮고, 소수가 선택한 팀(언더독)의 배당률이 높다. 배팅 참여 시점에 따라 실시간으로 배당률이 변동하며, 배팅 확정 시점의 배당률이 아닌 경기 마감 시 최종 배당률로 정산된다.
배당 표시 규칙
실시간 배당률픽 카드 내 선택지별 배당률 표시 (예: ×2.3 / ×1.4). 배팅 풀 실시간 반영.
최종 배당률경기 마감 후 더 이상 변경 없음. 정산 기준 배당률로 고정 표시.
배당률 최솟값×1.0 (이론값). 실질적으로 ×1.0 이하 배당률은 발생 불가 (풀 전체 배분 구조).
소수점 처리배당률 소수점 2자리 표시 (예: ×2.33). 포인트 정산 시 소수점 이하 버림.

CASE A
수량 캡 방식 — 주간 재고 선착순
미결 · 검토 중

경품별 주간 수량을 고정 설정한다. 포인트 차감 후 수량 내 선착순으로 당첨 처리되며, 수량 소진 시 해당 경품 응모가 마감된다. 잔여 수량을 화면에 노출하여 희소성 긴박감을 유발한다.

비용 제어 구조
주간 예산수량 × 경품 단가. 완벽히 예측 가능.
수량 리셋매주 월요일 00:00 초기화.
잔여 수량 표시"N개 남음" 실시간 노출. 소진 시 "이번 주 마감" 상태 전환.
당첨 확률 표시확률 미표시 (선착순 구조이므로 불필요). 단, "이번 주 N개 제공" 수량 공개.
장단점
장점단점
예산 완벽 통제. 월 비용 = 수량 합계 × 단가. 포인트를 모아온 유저가 품절로 사용 불가 시 이탈 유발.
"N개 남음" UI가 긴박감과 재방문 유인 생성. 초기 접속 유저가 재고 독점. 늦게 접속한 유저 박탈감.
SOOP 동일 모델 — 운영 선례 있음. 재고 관리 어드민 기능 별도 구현 필요.
선택 기준
💡 경품을 "빠르게 소진되는 인기 경쟁 이벤트"로 포지셔닝하고 싶을 때. 초기 유저 수가 적어 재고가 잘 소진되지 않으면 긴박감 효과가 반감될 수 있음.
CASE B
확률 기반 방식 — 고정 당첨 확률
미결 · 검토 중

경품별 고정 당첨 확률을 설정한다. 수량 제한 없이 포인트만 있으면 언제든 응모 가능하며, 응모마다 독립 확률이 적용된다. 품절 상태가 없어 유저 실망 포인트가 없다.

비용 제어 구조
주간 예산총 응모 횟수 × 확률 × 경품 단가. 유저 행동에 따라 변동.
확률 조정MAU 증가 또는 예산 초과 시 확률값 낮춰 어드민 조정. 즉시 반영.
당첨 확률 표시법적 의무로 응모 전 확률 명시 필수. (예: "당첨 확률 약 N%")
품절 상태없음. 항상 응모 가능.
장단점
장점단점
품절 없음 — 포인트 보유 유저 언제든 시도 가능. MAU 급증 시 예산 초과 위험. 사전 모니터링 필요.
운영 단순. 재고 관리 불필요. 확률값만 관리. 낮은 확률이 화면에 노출되면 참여 유인 저하 가능.
유저 수 증가에 따라 확률 조정으로 유연하게 대응. 누적 당첨 수가 매주 달라져 예산 예측이 어려움.
선택 기준
💡 경품을 "언제든 공정하게 도전 가능한 기회"로 포지셔닝하고 싶을 때. 초기 유저 수가 적은 지금 예산 변동 폭이 작아 관리 가능. 서비스 성장 후 확률 조정으로 지속 운영.
당첨 처리 (공통)
기프티콘 발급당첨 즉시 기프티콘 코드 인앱 발급. 마이 > 내 지갑 > 당첨 내역에서 확인.
발급 유효기간발급일로부터 30일. 이후 자동 소멸 (기프티콘 발행사 정책 준용).
발급 실패 처리오류 시 포인트 즉시 복구 + 재시도 안내. 재시도 실패 시 CS 자동 접수.
1인 수령 한도월간 총 수령 기프티콘 가치 한도 설정 필요 (경품류 고시 준수). 수치 미확정.

⛔ 허용하지 않는 소진 방식: 현금 환전, 응모권 전환, 타 유저 이전, 외부 서비스 연동. 포인트는 위 두 가지 용도로만 사용 가능.
05
포인트 정책
유효기간 및 소멸
기본 유효기간KBO 시즌 종료 후 30일. 매년 11월 말 기준 일괄 소멸 예정.
비시즌 대응마지막 서비스 활동일 기준 90일 미사용 시 소멸. KBO 시즌 유효기간보다 우선 적용.
소멸 사전 안내소멸 14일 전 앱 푸시 알림 발송. "포인트가 N일 후 소멸 예정입니다."
소멸 후 복구불가. 소멸 완료 포인트는 복구하지 않음.
이전·양도·환금 금지
계정 간 이전금지. 포인트는 획득 계정에만 귀속.
현금 환전금지. 어떠한 방식으로도 현금화 불가.
응모권 전환금지. 포인트 ↔ 응모권(TICKET) 교환 불가.
계정 탈퇴 시탈퇴 즉시 잔여 포인트 전액 소멸. 복구 불가.
서비스 종료 시종료 30일 전 공지 후 미사용 포인트 소멸.
어뷰징 방지
다중 계정동일 기기/CI 기준 중복 포인트 획득 차단. 어드민에서 의심 계정 플래깅 가능.
RV 광고 어뷰징기존 일일 5회 한도 유지. 광고 SDK 콜백 수신 기준으로만 포인트 지급.
배팅 풀 조작비정상적으로 높은 배팅 편중 감지 시 어드민 알림. 배당 왜곡 방지.
계정 거래서비스 약관 위반으로 처리. 적발 시 계정 정지 및 포인트 몰수.
이용약관 연동 필수 고지 사항
서비스 약관 및 포인트 관련 화면에 아래 항목을 명시적으로 고지해야 한다.

① 포인트는 폴리볼 서비스 내 활동을 통해 무료로 제공되는 가상 재화이며 금전적 가치가 없습니다.
② 포인트는 현금, 다른 재화, 다른 계정으로 이전하거나 교환할 수 없습니다.
③ 경품 응모는 확률에 의해 결정되며 당첨을 보장하지 않습니다.
④ 미사용 포인트는 [유효기간] 경과 후 소멸되며 복구되지 않습니다.
⑤ 부정 취득 또는 약관 위반 적발 시 포인트 회수 및 계정 이용 제한될 수 있습니다.

예외 처리
경기 취소 / 우천 순연
상황처리 방식포인트 처리
경기 취소 (우천·기타) 해당 경기 마켓 전체 CANCELED 상태 전환 배팅 포인트 전액 반환 (참여 보상 30P/15P는 유지)
경기 순연 (다음 날 진행) 배팅 마감 시각 연장 처리. 어드민 수동 조정. 포인트 유지 (기존 배팅 그대로 유효)
경기 무효 (5회 미만 경기 등) 취소 처리와 동일 배팅 포인트 전액 반환
결과 번복 (오심 정정) 어드민 정산 취소 → 재정산 실행 잘못 지급된 포인트 회수 후 올바른 배당으로 재지급. 유저에게 개별 알림.
서버 오류 / 배팅 처리 실패
상황처리 방식
배팅 확정 중 서버 오류 트랜잭션 롤백. 포인트 차감 없음. 유저에게 "잠시 후 다시 시도해 주세요" 안내.
배당 정산 중 오류 정산 일시 중단. 어드민 알림. 복구 후 재정산 실행. 최대 1시간 내 재처리.
포인트 지갑 동기화 실패 서버 기준 잔액 표시 (로컬 캐시 무효화). 새로고침 유도.
경품 응모 중 네트워크 단절 응모 요청이 서버에 도달하기 전 단절 시: 포인트 차감 없음, 응모 미처리.
서버 처리 후 단절 시: 서버 상태 기준. 유저 앱 재진입 시 결과 표시.
기프티콘 발급 실패 당첨 기록 보존 후 포인트 즉시 복구. 재시도 버튼 제공. 2회 이상 실패 시 CS 자동 접수.
경품 수량 소진 처리
응모 진행 중 수량 소진응모 요청이 서버에 도달했을 때 수량이 0이면 → 포인트 차감 없음, "해당 경품이 모두 소진됐어요" 안내. 응모 미처리.
동시 응모 처리수량 차감은 선착순 트랜잭션 처리. 동시 요청 시 먼저 처리된 요청만 당첨 가능.
소진 후 UI해당 경품 카드 "이번 주 소진 완료" 상태로 전환. 다음 주 리셋 안내.
포인트 과다 지급 오류
발생 원인정산 버그, 어드민 오설정 등으로 비정상 포인트 지급 시
처리 방식어드민에서 유저별 포인트 회수 가능. 회수 시 유저에게 푸시 알림 + 인앱 메시지 발송.
이미 소진된 경우소진 불가 회수분은 마이너스 처리하지 않음. 향후 획득 포인트에서 상계 처리 (어드민 설정).
