01배경 및 목적
기능 도입 배경

폴리볼은 현재 승부 예측과 응모 외에 유저가 서비스에 머물 이유가 부족하다. 예측 유저 1,487명(주간 기준) 중 응모로 전환되지 않는 유저가 약 36%에 달하며, 이 유저들을 서비스에 묶어두는 추가 가치가 필요하다.

동시에 KBO 직관 기록 시장(야구보구, 직관일기 등)에서는 서버 불안정으로 인한 데이터 유실이 핵심 이탈 원인으로 확인된다. 기업 수준의 인프라와 승부 예측 데이터가 결합된 기록 경험에 대한 니즈가 존재한다.

1차 목적 — 리텐션
📌 이 기능의 1차 목적은 리텐션이다. 기록이 쌓일수록 유저가 폴리볼을 떠나기 어려운 구조를 만드는 것이 핵심이며, 바이럴과 신규 유저 획득은 2차 목적이다. BEP 달성이라는 최종 목표와의 연결 경로는 다음과 같다.
경로흐름현재 단계
리텐션기록 축적 → 재방문 증가 → 광고 노출 → 매출1차 목적
바이럴카드 공유 → SNS 유입 → 신규 유저 → 광고 노출 → 매출2차 목적
Non-Goals
  • ❌기록의 진위를 기술적으로 100% 증명하는 것. 기록 자체에 경제적 보상이 없으므로 어뷰징 동기가 약하며, 기록 진위는 유저 양심에 맡긴다.
  • ❌복잡한 인증 허들로 기록 진입장벽을 높이는 것. 기록량이 쌓여야 소셜 확산과 BM이 작동한다.
  • ❌폴리볼 내부 커뮤니티 기능 신규 구축. 외부 SNS 공유로 대체한다.
  • ❌집관 기록의 카드 공유. 집관은 리텐션용 라이트 기능으로만 제공한다.
02핵심 정의 재정립
신뢰도란 무엇인가

v0.2까지는 신뢰도를 "기록이 진짜인지 확인하는 것"으로 정의하고 기술적 인증 방식을 탐색했다. v0.3에서는 신뢰도를 두 가지로 재정의한다.

신뢰도 유형정의폴리볼의 대응
보존의 신뢰내가 남긴 기록이 사라지지 않고, 내가 남긴 시점 그대로 보존된다기업 수준의 클라우드 인프라. 타 서비스 유실 문제 해결이 핵심 차별점
예측 데이터 연결경기 전에 이미 예측에 참여했다는 데이터가 폴리볼 안에 존재한다"이 기록은 경기 전부터 이 경기에 관심을 갖고 있던 사람이, 경기가 끝난 직후에 남긴 기록"이라는 문장을 폴리볼만 줄 수 있다
📌 기록의 진위(실제로 직관 갔는지)는 유저 양심에 맡긴다. 레터박스드, 굿리즈, 스트라바 모두 자기신고 방식으로 운영된다. 기록에 경제적 보상이 연계되지 않는 한 어뷰징 동기는 약하다. 경제적 가치 연계(NFT 발행, 보상 연동 등) 시점에 인증 체계 도입을 재검토한다.
카드란 무엇인가 — NFT 형태의 의미

카드가 NFT 형태인 이유는 두 가지다. 희소성(이 카드는 세상에 하나뿐이다)과 소유권(이건 내 것이다)이다. 이 두 가지는 예측 이력 없이도 카드 자체의 구조에서 나온다.

카드 가치 레이어내용예측 필요 여부
기본 가치그 경기, 그 날의 유일한 기록. 고유 발행 번호로 소유권 확인불필요
서사 레이어경기 전 예측 결과가 카드에 연결되어 이야기가 생김있으면 추가
희소성 극대화당첨 직관, 역사적 경기(한국시리즈, 끝내기 등)불필요
📌 예측 연동은 카드의 존재 이유가 아니라 카드에 서사를 더하는 레이어다. 예측에 참여하지 않은 유저도 카드를 만들 수 있어야 한다.
직관 vs 집관 역할 분리
구분기능 목적카드 발행공유
직관 기록자랑 소재 + 리텐션O — 공유 가능 카드O
집관 기록리텐션 전용X 또는 낮은 등급 카드X

집관은 "내가 끝까지 챙겨봤다"는 기록이지만 직관처럼 "내가 거기 있었다"는 특별한 경험이 아니다. 카드의 가치를 희석하지 않기 위해 집관은 리텐션용 라이트 기능으로만 제공하며, 프로필 통계에는 반영한다.

05감정선 문구

카드에 표시되는 대표 문구는 예측 결과와 경기 결과의 조합으로 자동 분기된다. 결과를 "맞다/틀리다"로 표현하지 않고 팬의 감정 언어로 표현하여 공유 욕구를 높인다.

예측 ✅ + 직관 승
"예측도 맞추고,
직관도 이겼다" 🔥
최고조의 감정. 자랑 욕구가 가장 강한 케이스. 공유 확률 최고.
조건: 예측 참여 O + 예측 적중 O + 경기 승리
예측 ✅ + 직관 패
"내가 이럴 줄
알았지" 😏
아이러니한 감정. "질 거라고 예측했는데 역시나"의 자조적 자랑. 공유 욕구 존재.
조건: 예측 참여 O + 예측 적중 O + 경기 패배
예측 ❌ + 직관 승
"예측은 빗나갔지만,
직관은 이겼다" 🎉
안도와 기쁨. 예측이 틀렸어도 결과가 좋아서 공유 가능.
조건: 예측 참여 O + 예측 적중 X + 경기 승리
예측 ❌ + 직관 패
"오늘은 졌지만,
기록은 남긴다" ⚾
위로가 필요한 감정. 공유는 안 하더라도 기록은 남기고 싶음. 리텐션 핵심 케이스.
조건: 예측 참여 O + 예측 적중 X + 경기 패배
⚠️ 예측에 참여하지 않은 경우: 감정선 문구 없이 경기 결과만 표시. 예측 참여 유도 넛지를 카드 하단에 노출하는 방안 검토 필요. TBD
06프로필 카드

07자랑 구조

유저가 자랑하는 조건은 두 가지다. 내가 특별한 경험을 했을 때, 그리고 내가 남들보다 잘했을 때. 폴리볼 카드는 이 두 가지를 모두 제공할 수 있다.

레이어 1
카드 디자인 — 올리고 싶게 생겨야 한다 팀 컬러, 경기 결과, 감정선 문구가 한눈에 보이는 카드. 못생기면 아무도 올리지 않는다. 여성 관람객 비율 56.7% 기반 포토카드 수집 감성 우선 설계.
레이어 2
숫자 — 비교와 자랑의 소재 프로필 카드의 통계 숫자(직관 횟수, 승률, 예측 적중률)가 비교 욕구를 만든다. "나 직관 승률 75%야"는 숫자가 있어야 자랑이 된다.
레이어 3
타이밍 — 흥분 상태를 잡아야 한다 경기 종료 후 푸시 알림으로 카드 완성 유도. 흥분 상태일 때 공유 확률이 높다. 시간이 지나면 식는다. 기본값 경기 종료 1시간 후, 유저 설정 가능.

#항목내용담당
1집관 기록 카드 등급 설계카드 없음 vs 낮은 등급 카드 중 결정. 낮은 등급으로 간다면 등급 기준과 시각 처리 방식 정의 필요.기획 + PD
2예측 미참여 직관 기록 문구예측에 참여하지 않은 경우 감정선 문구 없이 경기 결과만 표시할지, 예측 참여 유도 넛지를 카드에 포함할지 결정.기획
3카드 디자인 방향팀 컬러, 포토카드 감성, 배경 커스터마이징 등 상세 디자인 가이드 별도 작업 필요.PD
4프로필 카드 공유 시 노출 범위공유 시 닉네임 외 어떤 정보까지 외부에 노출할지 정의. 개인정보 최소화 원칙 적용.기획 + 법무
5KBO 데이터 연동 법무 검토공식 API 여부 및 크롤링 가능 여부 확인. 결과에 따라 자동 완성 범위 결정. 착수 전 필수.법무 + 개발
6사진 업로드 장수 제한 정책무료 제공 범위 및 유료 포토패스 기준 정의.기획 + 비즈니스
7내 지갑 IA 위치 변경마이 메뉴 내 현재 위치 확인 및 우선순위 상향 범위 정의.UX + 기획
8인증 체계 재검토 시점경제적 가치 연계(NFT 발행, 보상 연동 등) 단계에서 OCR/GPS 인증 도입 여부 재검토.기획 + 법무
