# 영역 A — 직관/집관 기록

---

## S-01 기록 작성 진입

**화면 개요**
직관/집관 기록을 시작하는 두 가지 경로를 정의한다. 유저가 기록을 시작하는 방법은 경기 종료 후 앱이 자동으로 알림을 발송하는 푸시 기반 진입과, 유저가 능동적으로 기록 탭에서 시작하는 수동 진입이다. 두 방식은 동일한 기록 작성 화면으로 연결되며, 진입 경로에 따라 사전 입력 항목의 자동 완성 여부가 달라진다.

> **⚠️ TBD:** 진입 방식 범위(푸시/수동/혼합) 확정 필요

---

### 진입 경로 A — 푸시 알림 기반 (자동 트리거)

경기가 종료되는 시점에 폴리볼 서버가 해당 경기와 연관된 유저에게 푸시 알림을 발송한다. 알림 대상은 해당 경기에 승부 예측 참여 이력이 있는 유저를 기본으로 하며, 향후 GPS 인증 유저(고도화 버전)로도 확장 가능하다.

**플로우**

| 단계 | 내용 | 비고 |
|---|---|---|
| 01 | 경기 종료 → 서버가 예측 참여 유저에게 푸시 발송 | 문구 예시: "오늘 LG vs 두산 경기가 끝났어요. 직관/집관 기록을 남겨보세요 ⚾" |
| 02 | 유저가 푸시 탭 → 앱 실행 후 기록 작성 화면으로 딥링크 이동 | 앱 비활성 시 실행 후 해당 화면으로 직접 진입 |
| 03 | 기록 작성 화면 진입 시 해당 경기 정보 자동 완성 상태로 노출 | |
| 04 | 직관 여부 선택 UI 노출 — "오늘 직접 가셨나요, 집에서 보셨나요?" | 선택에 따라 S-02, S-03으로 분기 |

> **⚠️ 주의:** 푸시 발송 시점(경기 종료 후 즉시 vs 딜레이)은 별도 알림 기획 문서에서 정의한다. 유저가 푸시를 무시한 경우 기록은 생성되지 않으며, 이후 수동 진입으로 소급 가능하다.

---

### 진입 경로 B — 수동 진입 (능동 트리거)

유저가 직접 기록 탭 또는 내 지갑에서 기록 남기기 버튼을 탭하여 시작하는 방식이다. 푸시를 놓쳤거나 과거 경기를 기록하고 싶은 경우에도 사용된다.

**플로우**

| 단계 | 내용 | 비고 |
|---|---|---|
| 01 | 기록 탭 또는 내 지갑에서 "+ 기록 남기기" 버튼 탭 | |
| 02 | 날짜 선택 → 해당 날짜의 KBO 경기 목록 자동 노출 | 경기 없는 날짜 선택 시 "해당 날짜에 경기가 없어요" 안내 |
| 03 | 경기 선택 → 직관/집관 여부 선택 → 기록 작성 화면 진입 | 선택한 경기 정보 자동 완성 상태로 노출 |

> **📌 참고:** 두 진입 경로 모두 동일한 기록 작성 화면(S-02, S-03)으로 연결된다. 차이는 직관/집관 선택 UI의 노출 시점과 형태에 있다.

---

### 직관 / 집관 선택 UI

| 항목 | 내용 |
|---|---|
| 선택 문구 | "오늘 경기, 어디서 보셨나요?" |
| 선택지 | 🏟️ 직접 갔어요 (직관) / 📺 집에서 봤어요 (집관) |
| 노출 시점 | 푸시 진입 시 기록 작성 화면 최상단 / 수동 진입 시 경기 선택 직후 |
| 선택 후 동작 | 직관 선택 → S-02 / 집관 선택 → S-03 |
| 언어 톤 | "직관/집관" 단어 대신 일상 언어로 표현. 라이트 유저 진입 장벽 최소화 |

## S-02 직관 기록 작성 화면

**화면 개요**
직관(현장 관람)을 선택한 유저가 기록을 작성하는 화면이다. 경기 기본 정보는 자동 완성된 상태로 제공되며, 유저는 선택 항목만 추가 입력하면 된다. 아무것도 추가하지 않아도 기록이 완성된 것처럼 느껴져야 하며, 추가 입력은 기록을 풍성하게 만드는 선택지로 제시된다.

**진입:** S-01 → **이동:** S-04

---

### 자동 완성 영역

유저 입력 없이 시스템이 자동으로 채우는 항목이다. 화면 상단에 카드 형태로 노출되며 수정이 불가하다.

| 항목 | 데이터 출처 | 표시 방식 | 비고 |
|---|---|---|---|
| 경기 날짜 | 진입 시점 또는 유저 선택 | YYYY.MM.DD (요일) | |
| 구장명 | KBO 경기 데이터 | 구장 전체 명칭 | |
| 홈팀 / 원정팀 | KBO 경기 데이터 | 팀명 + 홈🏠 아이콘 | |
| 최종 스코어 | KBO 경기 결과 데이터 | 홈팀 N : N 원정팀 | 경기 종료 후 자동 반영 |
| 승패 결과 | 스코어 기반 자동 계산 | 승 / 패 / 무 텍스트 | |
| 예측 참여 결과 | 승부 예측 이력 | ✅ 예측 성공 / ❌ 예측 실패 | 예측 이력 없으면 미표시 |

> **⚠️ KBO 데이터 연동 불가 시 (플랜 B):** 날짜·팀은 유저 선택으로 자동 완성, 스코어는 유저 직접 입력. 연동 확보 후 자동화 전환.

---

### 선택 입력 영역

유저가 원할 때만 추가하는 항목이다. 입력하지 않아도 기록 저장이 가능하며, 각 항목은 독립적으로 추가/제거할 수 있다.

| 항목 | 입력 방식 | 상세 |
|---|---|---|
| 사진 | 갤러리 선택 또는 카메라 촬영 | 업로드 장수 정책 별도 정의. 초과 시 유료 포토패스 유도. |
| 한 줄 소감 | 텍스트 자유 입력 | 최대 글자 수 정의 필요. 플레이스홀더: "오늘 직관 한 줄로 남겨봐요" |
| 동행인 태그 | 텍스트 입력 | MVP에서는 텍스트만. 계정 연동은 추후 고도화. |
| 날씨 | 이모지 셀렉터 (단일 선택) | ☀️ 맑음 / ⛅ 흐림 / 🌧️ 비 / ❄️ 추움 / 🌡️ 더움 |
| 기분 | 이모지 셀렉터 (단일 선택) | 😊 좋음 / 😐 보통 / 😢 아쉬움 / 🎉 최고 / 😤 짜증 |

## S-03 집관 기록 작성 화면

**화면 개요**
집관(집에서 시청)을 선택한 유저가 기록을 작성하는 화면이다. S-02와 구조는 동일하나, 사진 항목의 우선순위가 낮아지고 선택 입력 항목이 보다 가볍게 구성된다. 집관의 본질은 "이 경기를 진심으로 챙겨봤다"는 팬심의 기록이므로, 부담 없이 빠르게 완성할 수 있는 플로우가 핵심이다.

**진입:** S-01 → **이동:** S-04

---

### S-02와의 차이점

| 항목 | 직관 (S-02) | 집관 (S-03) |
|---|---|---|
| 자동 완성 카드 | 날짜·구장·팀·스코어·예측결과 | 날짜·팀·스코어·예측결과 (구장 미표시) |
| 카드 배경 텍스처 | 구장 분위기 | 홈/실내 분위기 (구분 필요) |
| 사진 항목 | 우선순위 높음 | 우선순위 낮음 (하단 배치 또는 생략 고려) |
| 추가 입력 항목 | 사진·소감·동행인·날씨·기분 | 소감·기분 (날씨·동행인 선택적 노출) |
| 기록 카드 표기 | "직관" 레이블 | "집관" 레이블 |

> **📌 참고:** 집관 화면은 직관 화면 기반으로 불필요한 항목을 걷어내어 더 빠르게 완성되는 경험을 제공한다. 단일 화면 분기 처리 vs 별도 화면 개발은 개발팀과 협의 필요.

---

### 집관 인증 처리 방식

GPS 인증 없이 집관 기록을 생성하는 MVP에서는 별도 인증 절차 없이 유저의 자기 선언으로 처리한다.

| 조건 | 처리 |
|---|---|
| 예측 참여 이력 있음 | 카드에 "✅ 예측 참여" 자동 표기. 간접 인증 근거로 활용. |
| 예측 참여 이력 없음 | 별도 표기 없이 일반 집관 기록으로 처리. |
| 보상 연동 시 | v2 시점에 실시간 퀴즈 인증 등 신뢰도 장치 별도 설계. MVP에서는 보상 미연동. |

## S-04 기록 카드 완성 화면

**화면 개요**
기록 저장 완료 직후 노출되는 화면이다. 단순한 완료 메시지가 아니라 완성된 카드를 실제로 보여주며 소유감을 형성한다. SNS 공유 또는 내 지갑 이동의 두 가지 액션으로 자연스럽게 연결된다.

**진입:** S-02 / S-03 → **이동:** S-08 내 지갑

---

### 화면 구성 요소

| 요소 | 내용 | 상세 |
|---|---|---|
| 완성 메시지 | "기록 카드가 생성됐어요 🎉" | 카드 생성 애니메이션과 함께 노출 (디자인 정의 필요) |
| 카드 미리보기 | 완성된 기록 카드 전체 노출 | 직관/집관/응모당첨 유형에 따라 카드 디자인 구분 |
| 고유 번호 | 카드 하단 발행 번호 | 예: #00847 · 형식은 디자인에서 정의 |
| SNS 공유 버튼 | 카드 이미지 저장 또는 링크 공유 | 이미지 저장 / URL 복사 / OS 공유 시트 |
| 내 지갑 이동 버튼 | "지갑에서 보기" | S-08 내 지갑으로 이동, 해당 카드 포커스 상태 진입 |
| 닫기 | 화면 종료 | 이전 화면 또는 홈으로 복귀 |

---

### 카드 공유 URL

## S-07 직관 승률 통계 화면

**화면 개요**
누적 기록을 바탕으로 개인화된 직관/집관 승률 데이터를 보여주는 화면이다. 기록이 적은 초반에는 통계를 바로 노출하지 않고, 5개 이상 쌓였을 때 공개되는 방식으로 설계한다. 오픈 전 프로그레스 바 자체가 기록 추가를 유도하는 리텐션 장치가 된다.

**진입:** 내 지갑 또는 기록 탭에서 진입

---

### 상태별 화면 정의

**잠금 상태 (기록 5개 미만)**
통계 수치 대신 프로그레스 바와 "기록 N개 더 쌓으면 내 승률이 열려요" 문구 표시. 통계 영역은 블러 또는 잠금 아이콘으로 처리하여 오픈 후 어떤 정보가 제공되는지 미리 보여준다.

**오픈 상태 (기록 5개 이상)**
통계 탭 오픈. 최초 오픈 시 "승률이 열렸어요! 🎉" 축하 연출 노출. 이후 진입 시 통계 바로 표시.

---

### 오픈 후 제공 통계 항목

| 항목 | 설명 | 표시 방식 |
|---|---|---|
| 직관 승률 | 직관 기록 중 응원팀 승리 비율 | % 숫자 강조 |
| 집관 승률 | 집관 기록 중 응원팀 승리 비율 | % 숫자 강조 |
| 직관 vs 집관 비교 | 두 승률 나란히 비교 | "집관 승률이 더 높아요" 등 코멘트 문구 추가 |

| 총 직관 횟수 | 직관 기록 전체 건수 | N회 |
| 총 집관 횟수 | 집관 기록 전체 건수 | N회 |

> **📌 주의:** 응원팀 설정 여부에 따라 승패 기준이 달라질 수 있음. 응원팀 미설정 유저의 승패 기준은 별도 정의 필요 (홈팀 기준 or 유저 직접 선택).
