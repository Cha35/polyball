1. 개요

본 문서는 마이 > 내 응모 > 응모 내역 화면에서 유저에게 노출되는 상태값의 정의, 전환 조건, 처리 방식을 규정합니다.

📌 본 문서의 상태값은 응모 내역 화면 전용입니다. 응모 메인의 경기 카드 상태(응모 진행 중 / 치열⚡ / 응모 마감)와는 별개입니다.

적용 화면 및 탭 구조

전체
응모 진행 중
발표 대기
당첨
미당첨

각 탭은 해당 상태의 응모 건만 필터링하여 노출합니다.

3. 상태 전이 규칙

3-1. 기준 주차(W) 정의

기호의미
W유저가 응모에 참여한 주차 (월~일)
W+1응모 마감 및 당첨자 발표 주차 (월요일 기준)
W+2예비당첨 최종 처리 및 강제 전환 기준 주차

3-2. 주차별 타임라인

W 월 ~ W 일
유저가 응모에 참여하면 내역에 응모 진행 중 상태로 노출됩니다.
W+1 월요일 00시
자동 배치 처리 — 응모 진행 중 → 발표 대기 전환.
당첨 결과 데이터가 생성되며, 유저는 앱 진입 후 직접 확인해야 합니다.
W+1 월 ~ W+1 일 (발표 대기 기간)
유저가 응모 내역 상세에서 결과 확인하기 CTA를 클릭하면 즉시 결과 상태(당첨 / 예비당첨 / 미당첨)로 전환됩니다.
이 기간 동안 확인하지 않아도 W+2 월요일 00시에 강제 전환됩니다.
W+2 월요일 00시
자동 배치 처리 — 발표 대기 상태 중 미확인 건 전체를 당첨 결과(당첨/예비당첨/미당첨)로 강제 전환합니다.
동시에, 예비당첨 건에 대해 최종 당첨자 명단 업데이트를 진행하여 예비>당첨 또는 예비>미당첨으로 최종 확정합니다.

3-3. 상태 전이 테이블

현재 상태 전환 트리거 전환 후 상태 처리 방식
응모 진행 중 W+1 월요일 00시 발표 대기 자동 배치
발표 대기 유저가 결과 확인하기 CTA 클릭 당첨 / 예비당첨 / 미당첨 유저 액션
W+2 월요일 00시 (미확인 시) 당첨 / 예비당첨 / 미당첨 자동 배치 (강제)
예비당첨 W+2 월요일 최종 당첨자 명단 확정 예비>당첨 / 예비>미당첨 자동 배치
당첨 / 예비>당첨 / 예비>미당첨 / 미당첨 — 종단 (변경 없음) —

4. 예비당첨 처리 정책

4-1. 예비당첨 개요

항목내용
예비 인원 기준최초 당첨 인원의 1배수 선정 (추후 2배수로 변경 예정)
예비 결과 확정 기준 시점W+2 월요일 최종 당첨자 명단 업데이트 시
유저에게 노출되는 안내 문구"기존 당첨자가 취소하면 예비 순번에 따라 개별 연락을 드려요."

4-2. 케이스 A — 예비 → 당첨 전환

항목내용
발생 조건최초 당첨자 중 포기 선언 또는 무응답자 발생 → 해당 티켓을 예비 순번 유저에게 인계 → 예비 유저의 당첨 수락 응답 완료
상태 전환 시점W+2 월요일 최종 당첨자 명단 확정 업데이트 시
응모 내역 상태 변경예비당첨 → 예비>당첨
티켓 노출상태 전환 완료 후 마이 탭 티켓 목록에 해당 티켓 노출 (준비 완료 상태)
유저 알림미정의 (추후 정의 필요)

4-3. 케이스 B — 예비 → 미당첨 전환

항목내용
발생 조건 ①최초 당첨자 전원 당첨 확정 완료
발생 조건 ②앞 순번의 예비자가 당첨을 먼저 확정하여 해당 유저에게 차례가 돌아오지 않은 경우
상태 전환 시점W+2 월요일 최종 당첨자 명단 확정 업데이트 시
응모 내역 상태 변경예비당첨 → 예비>미당첨
⚠️ 최초 당첨자 중 포기/무응답 유저의 취소 처리는 추후 취소 상태값으로 별도 반영 예정이며, 현재는 미당첨으로 임시 처리합니다.

5. 결과 확인 화면

발표 대기 상태의 응모 내역 카드에서 결과 확인하기 CTA 클릭 시 진입하는 화면입니다.

결과 화면 타이틀 본문 안내 문구 CTA 버튼 이동 화면
당첨 당첨 축하 화면 당첨 축하 메시지 티켓 확인하기 해당 티켓 상세
예비당첨 예비 당첨, 순서를 기다리고 있어요 "기존 당첨자가 취소하면
예비 순번에 따라 개별 연락을 드려요."		 ① 다른 경기 둘러보기 (Primary)
② 목록으로 가기 (텍스트)		 ① 응모 메인
② 내 응모 목록
미당첨 미당첨 안내 화면 아쉬움 안내 + 재도전 유도 다른 경기 둘러보기 응모 메인
예비당첨 / 미당첨 결과 화면 하단에는 {내 응모}로 이동 텍스트 링크를 공통 제공합니다.
