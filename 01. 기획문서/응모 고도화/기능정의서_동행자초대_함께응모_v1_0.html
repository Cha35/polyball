CI — 동행자 초대 (당첨자 측)
티켓 상세(MY-03) 내 동행자 초대 영역 · 3가지 상태
기존 화면 확장마이_기능정의서_v1.0의 티켓 상세(MY-03) 화면에 동행자 영역을 추가합니다. 기존 티켓 카드 하단, 경고 문구 상단에 배치됩니다.
CI-01 동행자 영역 — 상태별 정의
상태조건UI 구성CTA
미초대 당첨 확정 후 동행자 미초대 👥 아이콘 + "함께 갈 동행자를 초대해 보세요" + 카운터(0/N명) Primary 버튼 "동행자 초대하기" + 텍스트 링크 "다음에 할게요"
수락 대기중 초대 발송 후 동행자 미수락 ⏳ 아바타 + "초대 발송됨" + 주황 상태("수락 대기중") + 카운터(0/N명) "다시 보내기" 버튼 + "다른 사람 초대하기" 텍스트 링크
확정 동행자 수락 완료 ✅ 아바타 + 동행자 닉네임 + 녹색 상태("수락 완료") + "확정" 뱃지 + 카운터(N/N명) 없음 (변경은 교환 가능 전환 전까지 가능)
CI-01 동작 정의
  • CI-1동행자 영역은 티켓 상태 "준비중"부터 "교환 가능" 전환 전까지 노출. 서류 제출 여부와 무관하게 독립 표시
  • CI-2동행자 슬롯 수: 당첨 매수 − 1. 2매 당첨 → 1명, 3매 당첨(기아 3매 응모) → 2명
  • CI-3"다음에 할게요" 탭 시 동행자 영역 축소. 티켓 상세 재진입 시 축소 상태 유지하되, 동행자 영역은 계속 노출
  • CI-4"다시 보내기" 탭 시 기존 초대 링크와 동일한 공유 시트(CI-02) 재호출
  • CI-5"다른 사람 초대하기" 탭 시 기존 초대 취소 → 미초대 상태로 전환 → 새 초대 가능
  • CI-6동행자 변경: 교환 가능 전환(경기 2시간 전) 전까지 자유롭게 변경 가능. 횟수 제한 없음
  • CI-7교환 가능 전환 시점 + 동행자 미초대 → 인앱 팝업 1회 노출: "아직 동행자를 초대하지 않았어요. 지금 초대할까요?" + "초대하기" / "괜찮아요"
  • CI-8"괜찮아요" 선택 시 동일 팝업 재노출 안 함. 내부 재무 처리는 임직원 정보로 fallback
CI-02 초대 공유 시트
트리거"동행자 초대하기" 버튼 탭
공유 방식커스텀 공유 시트: 카카오톡 / 링크 복사 (기존 SH 공유 시트 패턴 재사용)
공유 메시지UXW 확정 필요 경기 정보(날짜·팀·구장) + 초대 안내 + 딥링크
딥링크 구조polyball.kr/invite?ticket_id={ticketId}&host={userId} — 레퍼럴 링크(/join?ref=)와 분리
공유 썸네일경기 정보 카드 형태. SVG 형식 (디자인 가이드 Share 컴포넌트 참조)
CI — 기아 3매 당첨 시 (동행자 2명)
기아 3매 응모 당첨 케이스동행자 슬롯 2개가 노출됩니다. 각 슬롯은 독립적으로 초대/수락/확정 상태를 관리합니다. 부분 확정(1명만 수락) 시 확정된 동행자만 표시, 미확정 슬롯은 초대 가능 상태 유지.

CA — 초대 수락 (동행자 측)
딥링크 진입 → 가입/로그인 → 본인인증 → 수락
동행자 유형별 플로우
유형플로우예상 스텝
비회원 (앱 미설치) 링크 수신 → 스토어 이동 → 앱 설치 → 회원가입 → 온보딩(응원팀 선택) → 본인인증 → CA-01 초대 수락 → 확정 7스텝
기존 회원 (본인인증 완료) 링크 수신 → 앱 진입 → CA-01 초대 수락 → 확정 3스텝
기존 회원 (본인인증 미완료) 링크 수신 → 앱 진입 → 본인인증 → CA-01 초대 수락 → 확정 4스텝
CA-01 초대 수락 화면
헤더
  • 타이틀: "직관 초대"
  • X 닫기 버튼
경기 정보
  • 원정팀 vs 홈팀
  • 날짜 · 시간 · 구장
  • 좌석 정보
초대 안내
  • "[닉네임]님이 함께 직관 가자고 초대했어요"
  • "수락하면 경기 정보를 앱에서 확인할 수 있어요"
하단 CTA
  • Primary 버튼: "초대 수락하기"
  • 본인인증 미완료 시: "본인인증 후 수락하기" (인증 화면 → 수락 자동 처리)
CA — 동작 정의
  • CA-1비회원 딥링크 진입 시 스토어로 이동. 앱 설치 + 가입 + 온보딩 완료 후 CA-01 화면으로 자동 랜딩 (딥링크 파라미터 유지)
  • CA-2온보딩은 기존 프로세스 그대로 수행 (응원팀 선택 포함). 스킵 불가
  • CA-3본인인증 필수. 휴대전화번호 수집 목적(애플 가입자 대응) + DI 기반 어뷰징 방지
  • CA-4"초대 수락하기" 탭 → 수락 완료 화면: "수락 완료! 경기 정보를 마이에서 확인해 보세요" + 마이 이동 CTA
  • CA-5수락 완료 시 당첨자(호스트)에게 푸시 발송: "동행자가 초대를 수락했어요" (CRM 섹션 참조)
  • CA-6신규 가입자 동행자에게 웰컴 응모권 5장 지급. 수락 완료 화면에서 안내: "환영해요! 응모권 5장을 드렸어요"
  • CA-7기존 가입자 동행자에게는 응모권 미지급
  • CA-8레퍼럴 보상 자동 지급 안 함. 레퍼럴 코드는 별도 경로(RF-01 ⑦)에서 수동 입력
  • CA-9이미 취소된 초대 링크 진입 시 안내 화면: "이 초대는 더 이상 유효하지 않아요" + 홈 이동 CTA

GC — 동행자 경기 정보 카드
동행자 마이 지갑 내 경기 정보 노출 (QR/난수 미노출)
GC-01 경기 정보 카드
노출 위치마이 > 내 지갑 영역. 기존 티켓 카드와 동일 위치, 다른 UI
노출 조건초대 수락 완료 후 ~ 경기 종료 + 3일 (기존 티켓 종료 정책과 동일)
카드 구성경기 정보(원정 vs 홈, 날짜, 시간, 구장) + "티켓은 [호스트 닉네임]님이 수령해요" 안내
QR / 난수번호미노출. 교환 주체는 당첨자(호스트)이므로 동행자에게 교환 정보를 제공하지 않음
카드 클릭경기 상세 정보 바텀시트. 경기 정보 + 교환 장소 + 호스트 닉네임 표시
상태 뱃지"직관 예정" (경기 전) / "직관 완료" (경기 후)
GC — 동작 정의
  • GC-1동행자 카드는 기존 티켓 카드와 시각적으로 구분. 호스트 티켓은 파란 헤더, 동행자 카드는 회색 또는 연보라 헤더로 차별화
  • GC-2함께 응모로 당첨된 경우 자동 생성. 당첨 후 초대로 수락한 경우에도 동일 카드 생성
  • GC-3호스트가 동행자를 변경한 경우, 기존 동행자의 카드 삭제 + "초대가 취소되었어요" 안내 필요

응모 상세 내 함께 응모 분기 · 초대 · 합산 · 보너스

"혼자 응모" 선택기존 응모 플로우 그대로. Stepper + 응모하기 CTA
"함께 응모" 선택TE-02 초대 화면으로 전환. 게스트 합류 전까지 응모 버튼 비활성
TE-02 함께 응모 초대
UI 구성"함께 응모할 친구를 초대해 보세요" 안내 + 공유 시트 CTA
공유 방식카카오톡 / 링크 복사 (CI-02와 동일 패턴, 다른 딥링크)

트리거게스트 합류 완료 후 자동 전환

보너스 계산합산 응모권 10장당 +1장. 소수점 이하 버림. 예: 합산 153장 → 보너스 15장 → 총 168장
호스트 Stepper호스트가 본인 사용량 조절 가능. 게스트 사용량은 게스트가 결정 (수정 불가)
CTA"응모하기" — 호스트만 최종 응모 실행 가능
TE — 동작 정의
  • TE-1함께 응모 완료 시 호스트 · 게스트 각각에게 응모 완료 화면 노출 + RV 광고 노출. 기존 1일 5회 한도는 각 유저별 독립 적용
  • TE-2함께 응모 당첨 시 호스트 이름으로 처리. 티켓은 호스트 지갑에만 적재. 게스트는 동행자 자동 확정 (당첨 후 초대 프로세스 스킵)
  • TE-3함께 응모 미당첨 시 호스트 · 게스트 모두에게 미당첨 결과 안내
  • TE-4게스트 합류 전 호스트가 "취소" 시 혼자 응모 모드로 복귀
  • TE-5팀 응원전 점수: 호스트 · 게스트 각각 1회 응모로 카운트하여 점수 적립
  • TE-6함께 응모 시 매수 분기: 2매 경기 → 2인 팀, 기아 3매 경기 → 3인 팀(호스트 + 게스트 2명)
