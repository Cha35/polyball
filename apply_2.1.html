개요 및 BM 연결
왜 만드는가

직관 티켓 외 두 번째 보상 경로로 기프티콘·실물경품을 도입하고, 지급된 경품을 유저가 인앱에서 확인·수령할 수 있는 화면을 정의한다. 보유 경품은 마이 > 내 지갑의 보유 티켓 하단에 노출하고, 수령 진입은 마이 > 내 활동 > 응모 현황 > 경품 응모 및 당첨 알림에서 시작한다.

핵심 원칙 (v1.1)
  • 수령 모델 — 당첨 = 즉시 발급이 아니다. 기프티콘은 발행분 정산 대상이므로, 유저가 [받기]를 클릭한 시점에 발급/코드 할당한다.
  • 진입점 분리 — 수령은 경품 응모 현황 + 당첨 알림에서 시작한다. 내 지갑은 수령 완료된 경품만 노출한다(수령 전 건은 지갑에 넣지 않음).
  • 상세 진입 조건 — 코드(바코드/핀)가 있는 상세는 발급 완료(사용가능) 이후에만 진입한다. 발급 전 '빈 상세'는 존재하지 않는다.
BM 연결
고정 공급되는 직관 티켓은 유저 증가·시즌 종료 국면에서 당첨 희소성이 구조적으로 악화된다. 기프티콘·실물경품은 이 희소성을 분산하는 두 번째 보상 경로로, 픽/응모 지속 동기를 유지 → 광고 노출 방어에 기여한다. 보상 모델 검증 트랙의 인앱 표시 레이어에 해당하므로, 이 화면 자체에는 광고를 삽입하지 않고 마찰 없는 수령 경험을 우선한다.

제세공과금 = 운영위탁 (v1.2) — 제세공과금(5만원 초과 경품)의 신고 대행은 폴리볼이 아닌 운영위탁 업체가 진행한다(직관 티켓과 동일 구조). 폴리볼은 제세공과금 정보를 수집·입금 확인하지 않으며, 배송지 정보를 위탁업체에 전달하고 유저에게 "3영업일 내 제세공과금 관련 안내가 진행됩니다" 노티만 제공한다. 기프티콘은 제세공과금 발생 수준의 고가 상품을 운영하지 않는 것을 전제로 한다.
수령 모델 (정산 구조 반영) — 당첨 = 즉시 발급이 아니다. 기프티콘은 발행분 정산 대상이므로 무조건 선발급하지 않고, 당첨 결과 확인·[받기] 시점에 발급/코드 할당한다. 실물경품은 [배송지 입력]으로 수령 의사를 받는다. 수령 기한은 당첨 발표일로부터 7일이며, 기한 내 미수령 시 당첨이 자동 취소된다(복구 불가). (어드민 PZ는 '발급 예약'만 보유)

3-1. 진입점

수령은 내 지갑이 아니라 아래 두 경로에서 시작한다. 내 지갑은 수령 완료분만 보유하므로 수령 전 건의 진입점이 될 수 없다.

진입점역할
당첨 알림(알림톡)발표 시 발송(직관 티켓 패턴 재사용). 당첨 → 받기 동선으로 연결, 미당첨은 중립·재참여 유도 카피.
경품 응모 현황마이 > 내 활동 > 응모 현황 > 경품 응모 탭. 알림을 놓쳐도 결과·수령을 확인하는 영속 화면.
영속성 — 이벤트 페이지(응모하는 곳)는 종료·소멸 가능하지만, 경품 응모 현황은 유저↔응모/당첨 데이터 기반이라 페이지 소멸과 무관하게 내 내역·수령이 유지된다. 정규 경품 응모도 동일 세션 위에 얹는다.
3-2. 경품 응모 현황 화면

위치: 마이 > 내 활동 > 응모 현황 상단에 [직관 티켓 | 경품 응모] 탭 추가(기존 IA·화면 재사용). 데이터 단위는 경품 단위 — 스페셜 응모가 경품별로 응모되므로, 카드 = 내가 응모한 경품 1개.

카드 상태표시액션
응모 진행 중진행 중 (응모 기간)없음
발표 대기"발표됨" + [결과 확인하기] (직관 티켓 패턴 재사용)[결과 확인하기] → 결과 확인 화면(3-2a)
미당첨뱃지 + 딤수령 기한 내 [광고 보고 볼 받기](3-2b) → 기한/3회 소진 후 종결
당첨 · 수령 전당첨 뱃지 + 수령 기한 D-N (발표+7일)[경품 받기](기프티콘) / [배송지 입력](실물) → 수령 동선(3-3)

실물 '확인 중' = TBD — 5만원 초과 실물경품의 제세공과금은 위탁업체가 안내·처리하며, 접수 완료 화면·카드에 "3영업일 내 안내" 노티만 제공한다. 위탁업체가 처리 완료를 폴리볼에 통보할 수 있는 시점·방식이 확인되면 '확인 중 → 접수 완료' 상태 전환 도입을 결정한다.

3-2a. 결과 확인 동선

3-2b. 낙첨자 광고 시청 볼 환급

항목정책
환급 기준경품 단위 — 해당 낙첨 경품에 사용한 볼 수 기준 (회차 전체 아님)
최대 시청 횟수낙첨 경품 1건당 최대 3회 (계정 일괄 아님, 카드별 귀속)
회당 환급량round_up(사용 볼 × 10%), 최소 1볼 보장. 예) 30볼→3볼, 11볼→2볼, 9볼→1볼
시청 가능 기한발표일로부터 7일 (당첨 수령 기한과 동일 정책, 공용 리마인드 가능)

BM 연결 — 낙첨은 지금까지 완전한 데드엔드였다. 광고 시청을 매개로 볼을 환급하면 가장 이탈 위험이 큰 순간을 RV 시청 기회로 전환하고, 환급된 볼은 재응모로 돌아와 픽/응모 지속 동기에 재기여한다. 낙첨자 수가 당첨자보다 압도적으로 많아 RV 인벤토리 확대 효과가 크다.

4-1. 내 지갑 구성 순서 (변경)
본인인증 배너 → 보유 티켓 → [신설] 보유 경품 → 보유 응모권

기프티콘·실물경품은 별도 섹션을 늘리지 않고 "보유 경품" 단일 영역에 통합하며, 리스트 아이템의 타입 뱃지(기프티콘 / 실물)로 구분한다.

4-2. 노출 범위 · 상태 분기
내 지갑은 수령 완료된 경품만 노출한다. 수령 대기·발표 대기 건은 지갑이 아닌 경품 응모 현황(§3)에서 다룬다. 따라서 지갑 카드는 기프티콘 사용가능/사용완료/만료, 실물 접수 완료(+제세공과금 확인 중) 상태만 존재한다.
상태노출
보유 0건 섹션 헤더 유지 + Empty(타입 A) — "아직 받은 경품이 없어요." (보유 티켓 빈 상태 패턴과 통일)
보유 N건 수령 완료 경품 카드 리스트. 사용가능 상단, 사용완료·만료 하단 정렬.

6-1. 전체 플로우
경품 응모 현황(§3) 당첨 카드 [배송지 입력] 또는 당첨 알림 → ① 개인정보 수집·이용 동의 (본인인증 목적) → ② 본인인증 진행 (MY-07, 미완료 시) → ③ 배송지 입력 화면 · 배송지 정보 입력 (수령인 / 연락처 / 주소) · 배송지 정보 수집·이용 동의 (체크) · 배송지 정보 제3자 제공(운영위탁 업체) 동의 (체크) → ④ 저장(접수) → 접수 완료 · [경품가 5만원↑] 접수 완료 화면·카드에 노티: "배송지 정보가 위탁업체에 전달되며, 3영업일 내 제세공과금 관련 안내가 진행돼요"
동의는 정보를 실제 수집하는 시점에 받는다. ①은 본인인증 목적으로 분리하고, 배송지 수집·이용 동의와 제3자 제공(위탁) 동의는 ③에서 입력과 함께 받는다. 제세공과금은 폴리볼이 수집·입금 확인하지 않으며 위탁업체가 신고 대행한다(직관 티켓과 동일 구조).
6-2. ③ 배송지 입력 화면 상세
요소내용
수령인명Text Field (필수)
연락처Text Field (필수, 숫자)
주소우편번호 검색 + 상세주소 (필수)
배송지 동의Checkbox (필수) — "배송지 정보 수집·이용 동의" + 전문 보기
제3자 제공 동의Checkbox (필수) — "배송지 정보 제3자 제공(운영위탁 업체) 동의" + 전문 보기. 배송·제세공과금 처리 목적 명시.
제세공과금 고지경품가 5만원 이상 시 안내 박스(고지, 동의 아님) — "5만원 이상 경품은 제세공과금 대상이며, 위탁업체가 3영업일 내 안내해 드려요"
CTA필수 입력·동의 완료 시 "신청 완료하기" 활성 (Primary)
수집·이용 동의와 제3자 제공 동의는 분리 체크박스로 받는다(개보법 §22 일괄 동의 금지). 제세공과금은 고지 사항으로만 표기하며 앱 내 정보 등록·입금 절차가 없다.
6-3. ④ 접수 완료
저장 즉시 접수 완료 + 배송지 요약 노출, 재진입 시 읽기 전용
[경품가 5만원↑] 접수 완료 화면·지갑 카드에 "3영업일 내 위탁업체가 제세공과금 안내" 노티 노출
접수 완료 = 폴리볼 측 종착 상태. 배송·송장·제세공과금 안내는 위탁업체 소관(배송 완료 정보는 폴리볼에 없음)
'확인 중' 상태 신설 여부는 TBD — 위탁업체의 처리 완료 통보 시점·방식 확인 후 결정

개보법 §15·§22배송지 수집·이용 동의필수 수집 시점(③ 배송지 입력)에 목적·항목·보유기간·거부권 고지. 배송지 동의와 제세공과금 동의를 분리 체크박스로 구성(일괄 동의 금지).
개보법 §24·§29본인인증 / 안전조치필수 본인인증 동의 목적에 '경품 수령'을 추가해야 함 — 현 레퍼런스는 "티켓 교환 목적에 한정"으로 기재됨. 수령인·연락처·주소는 암호화 저장.
소득세법제세공과금 (기타소득)5만원 초과 경품은 기타소득 신고 대상 — 운영위탁 업체가 신고 대행(직관 티켓과 동일). 폴리볼은 고지·노티만 담당.
개보법 §17·§26제3자 제공 / 처리위탁필수 배송지 정보를 위탁업체에 전달하므로 제3자 제공(또는 처리위탁) 동의·고지 필요. 직관 티켓과 동일 구조이므로 기존 위탁 계약·동의문 재사용 검토. 제공 항목·목적·보유기간 명시.
사행성/대가성 검토낙첨 광고 환급(볼)TBD 무료 재화(볼)의 사후 일부 환급이 경품 응모의 대가성·확률형 해석에 영향 없는지 법령 레퍼런스 점검 필요.
