## 3. 서버팀 데이터 요청 목록

### 3-1. 응모권 발급·소진 현황 🔴 최우선

**목적:** 예측은 하는데 응모를 안 하는 이유가 "응모권이 없어서"인지 "있는데 안 쓰는 것"인지 구분

```
요청 데이터:
- 일별 응모권 총 발급 수 (발급 사유별: 예측보상 / 광고시청 / 기타)
- 일별 응모권 총 소진 수
- 응모권 발급 후 미소진 유저 수 (잔고 > 0인 유저)
- 유저별 평균 응모권 잔고 (스냅샷: 매주 월요일 기준)

기간: 2026-03-26 ~ 2026-06-03 (일별)
```

**이걸로 볼 것:**
- 발급 대비 소진율 = 실제 "응모 의지" 지표
- 잔고 쌓이는 추이 = 유저가 응모권을 모으고 있는지 포기하고 있는지
- 월요일 전날(일요일) 잔고 vs 월요일 소진량 교차 → 가설 2 검증

---

### 3-2. 당첨/미당첨 후 행동 코호트 🔴 최우선

**목적:** 미당첨이 이탈의 직접 원인인지 확인 (어린이날 가설 검증)

```
요청 데이터:
- 응모 참여자 중 당첨/미당첨 결과별 이후 7일 내 재방문 여부
- 당첨자 vs 미당첨자의 이후 30일 응모 재참여율
- 특정 이벤트(어린이날 5/5~5/10) 응모자 코호트 재방문율

기간: 전체 응모 이력 기반 (결과 발표일 이후 추적)
```

**이걸로 볼 것:**
- 미당첨 후 재방문율이 당첨자 대비 얼마나 낮은가
- 어린이날 미당첨자의 이탈률이 다른 주차 대비 높은가
- → H1 가설 직접 검증

---

### 3-3. 유저별 앱 설치 여부 × 행동 지표 🟡 중요

**목적:** 앱 미설치 유저가 응모 루프(결과확인→재응모)에서 이탈하는지 확인

```
요청 데이터:
- 웹 가입 후 앱 설치까지 경과 일수 분포
- 앱 설치 유저 vs 미설치 유저의 예측·응모 전환율 비교
- 앱 설치 유저의 응모 횟수/유저 vs 웹 전용 유저

기간: 전체 가입자 기준 (가입일~2026-06-03)
```

**이걸로 볼 것:**
- "결과 확인 = 앱 필수"인데 앱 미설치 유저의 응모 재참여율이 낮다면 → H3 확인
- 앱 유저가 웹 유저보다 응모 횟수가 유의미하게 높다면 → 광고 루프 효과 확인

---

### 3-4. 응모권 발급 사유별 분해 🟡 중요

**목적:** 응모 횟수 증가가 광고 시청 때문인지 예측 보상 때문인지 구분

```
요청 데이터:
- 응모권 발급 사유: ① 예측 적중 보상 ② 광고 시청 ③ 이벤트/기타
- 사유별 발급량 일별 추이
- 광고 시청 유저 수 및 시청 횟수 (앱 전용)

기간: 2026-03-26 ~ 2026-06-03 (일별)
```

**이걸로 볼 것:**
- 광고 시청 유저가 응모를 더 많이 하는가 → H4 검증
- 광고 시청이 줄었다면 응모 횟수 감소와 연결

---

### 3-5. 리텐션 코호트 🟠 필요

**목적:** 가입 후 얼마나 오래 남아있는지, 어떤 시점에 이탈하는지

```
요청 데이터:
- 가입 주차별 D1 / D7 / D14 / D30 리텐션율
- 채널별(마케팅 채널) 리텐션 비교 (자리어때 유입 vs 인스타 유입 등)
- 어린이날 전후 가입자 코호트(4/14~4/27 vs 5/1~5/15) 리텐션 비교

형태: 코호트 테이블 (가입주차 × 경과주차)
```

**이걸로 볼 것:**
- 어느 가입 주차 유저가 빨리 이탈하는가
- 마케팅 채널별 유입 품질 차이 (CAC가 낮아도 리텐션이 낮으면 의미 없음)

---

### 3-6. 예측 완료 → 응모 도달 퍼널 (개인 수준) 🟠 필요

**목적:** "예측은 했는데 응모 안 한 유저"의 실제 이유 파악

```
요청 데이터:
- 예측 완료 유저 중 동일 경기 응모 참여 여부
- 응모권 잔고 있음 + 응모 미참여 유저 수 (일별)
- 응모 가능 경기가 있는 날 응모 미참여 유저의 행동 (예측만 하고 나간 경우)

기간: 2026-04-01 ~ 2026-06-03
```

**이걸로 볼 것:**
- 응모권 있는데 안 쓰는 유저가 실제 존재하는가
- 존재한다면 얼마나 되는가 → 응모 진입 장벽 UX 문제 가능성

## 5. 데이터 수령 후 분석 계획

```
Step 1. 응모권 발급/소진 데이터 확보
  → 발급 대비 소진율 트렌드 계산
  → 월요일 전후 소진 패턴 확인
  → "쌓아두기 vs 안 쓰기" 구분

Step 2. 당첨/미당첨 코호트 분석
  → 어린이날 응모자의 이후 2주 행동 추적
  → 당첨자 재응모율 vs 미당첨자 재응모율
  → 이탈 직접 원인으로서 미당첨 효과 측정

Step 3. 앱 설치 × 행동 교차 분석
  → 앱 설치 유저의 응모율이 유의미하게 높은지
  → 앱 설치 유도 시점과 응모 전환의 상관

Step 4. 통합 진단
  → 이탈 유형 분류:
     ① 응모권 소진 후 미당첨 이탈형
     ② 응모권 미사용 방치형
     ③ 예측만 하고 응모 무관심형
     ④ 신규 유입 부재형
  → 유형별 규모 추정
  → 유형별 대응 방안 제안
```
