## 서비스 로직 정의 (요청 작성 기준)

### 응모 방식 — 두 가지 타입

| 타입 | 명칭 | 조건 | 응모권 차감 | 광고 발생 |
|---|---|---|---|---|
| A | **응모권 응모** | 응모권 보유 한도 내 | 차감 O (1장/1회) | 없음 |
| B | **광고 응모** | 동일 경기 5회 응모 완료 후 광고 시청 | 차감 없음 (무료 처리) | 발생 O |

> 타입 B는 경기당 일 최대 5회 이후에만 활성화. 즉, 타입 B 이용자 = 동일 경기에 5회 이상 응모한 헤비유저

### 응모권 소진 정의
- **소진 = 타입 A 응모 완료 시** 응모권 1장 차감
- 타입 B(광고 응모) 완료 시에는 응모권 차감 없음

### 당첨 발표 시점
- **응모 기간 차주 월요일 오전 10시**
- 예: 4/20(월) 오픈 → 4/27(월) 10시 발표
- 예: 5/18(월) 오픈 → 5/25(월, 부처님오신날) 10시 발표

### 유저 식별 기준
- **UID 기준** (웹/앱 동일 유저 통합)

## 요청 1. 응모권 발급·소진·잔고 일별 집계

### 목적
"예측은 하는데 응모를 안 하는 유저"가 응모권이 없어서인지, 있는데 안 쓰는 건지 구분

### 전제: 타입A 응모권 사용 방식
유저가 1회 응모 시 응모권을 **n장 자유 선택하여 사용** (1장 고정이 아님)  
→ 응모 횟수와 소진된 응모권 수가 다름. 반드시 분리해서 집계 필요

### 지표 정의

**발급 관련**

| 지표명 | 정의 | 비고 |
|---|---|---|
| `issued_total` | 당일 발급된 응모권 총 수 (장) | 발급 이벤트 발생일 기준 |
| `issued_pred_reward` | 예측 적중 보상으로 발급된 수 | 발급 사유 = 예측 보상 |
| `issued_event_other` | 이벤트·프로모션 등 기타 발급 수 | 위 외 모든 발급 |
| `issued_users` | 당일 응모권을 1장 이상 발급받은 유니크 유저 수 | UID 기준 |

**소진 관련 (타입A — 응모권 차감 응모)**

| 지표명 | 정의 | 비고 |
|---|---|---|
| `entry_count` | 당일 타입A 응모 완료 횟수 | 유저가 응모 버튼을 누른 횟수 |
| `entry_users` | 당일 타입A 응모를 1회 이상 한 유니크 유저 수 | UID 기준 |
| `tickets_used_total` | 당일 타입A로 소진된 응모권 총 수 (장) | entry_count와 다름 — 1회 응모에 n장 사용 |
| `tickets_per_entry_avg` | 타입A 1회 응모당 평균 사용 응모권 수 | tickets_used_total / entry_count |
| `tickets_per_user_avg` | 당일 응모 유저 1인당 평균 소진 응모권 수 | tickets_used_total / entry_users |

**잔고 관련**

| 지표명 | 정의 | 비고 |
|---|---|---|
| `balance_users` | 당일 23:59 기준 잔고 > 0인 유니크 유저 수 | 누적발급 - 누적소진 > 0 |
| `balance_avg` | 잔고 보유 유저의 평균 잔고 | SUM(잔고) / balance_users |
| `balance_total` | 전체 유저 잔고 합계 | |
| `balance_not_entry_users` | 당일 잔고 > 0 이면서 응모를 하지 않은 유저 수 | balance_users 중 entry_users 제외 |
| `balance_not_entry_rate` | 잔고 보유자 중 미응모 비율 | balance_not_entry_users / balance_users × 100 |

> `balance_not_entry_rate`는 "응모권이 있는데 쓰지 않는 유저 비율"로, 응모 이탈의 핵심 진단 지표

### 출력 형식

```
date,
issued_total, issued_pred_reward, issued_event_other, issued_users,
entry_count, entry_users, tickets_used_total, tickets_per_entry_avg, tickets_per_user_avg,
balance_users, balance_avg, balance_total, balance_not_entry_users, balance_not_entry_rate(%)
```

### 기간
2026-03-26 ~ 2026-06-03 (일별, 70행)

## 요청 2. 광고 응모(타입 B) 현황 일별 집계

### 목적
광고 매출 급락의 원인 규명 — 헤비유저(경기당 5회 이상 응모자) 감소인지 확인

### 배경
타입 B 광고 응모는 "경기당 5회 응모 완료 후 광고 시청 시 1회 추가 응모 (무료 처리)" 방식.
광고 매출이 발생하는 구간이므로 이 지표의 감소 = 헤비유저 감소 = 광고 매출 감소

### 지표 정의

| 지표명 | 정의 | 비고 |
|---|---|---|
| `ad_entry_count` | 당일 타입B(광고 응모) 완료 횟수 | 광고 시청 후 응모 완료 건 |
| `ad_entry_users` | 당일 타입B를 1회 이상 사용한 유니크 유저 수 | UID 기준 |
| `ad_view_count` | 응모 목적 광고 시청 횟수 | 타입B 트리거 광고 노출 수 |
| `ad_view_to_entry_rate` | 광고 시청 후 응모 완료 전환율 | ad_entry_count / ad_view_count × 100 |
| `avg_ad_entry_per_user` | 타입B 사용 유저의 1인당 평균 광고 응모 횟수 | ad_entry_count / ad_entry_users |
| `heavy_users` | 당일 1개 이상의 경기에서 5회 이상 응모한 유저 수 | 타입B 자격 유저 풀 |

### 출력 형식

```
date, ad_entry_count, ad_entry_users, ad_view_count,
ad_view_to_entry_rate(%), avg_ad_entry_per_user, heavy_users
```

### 기간
2026-03-26 ~ 2026-06-03 (일별, 70행)

## 요청 3. 당첨/미당첨 결과 후 유저 행동

### 목적
미당첨 경험이 이탈의 직접 원인인지 확인

### 지표 정의

**집계 단위: 응모 결과 발표 회차별 (매주 월요일 10시)**

| 지표명 | 정의 | 비고 |
|---|---|---|
| `announce_date` | 당첨 발표일 (차주 월요일 10시) | |
| `result_type` | 당첨 / 미당첨 | |
| `user_count` | 해당 결과를 받은 유저 수 | UID 기준 |
| `revisit_d7` | 발표일 기준 7일 내 앱 or 웹 재방문 유저 수 | 예측·응모·퀴즈 이벤트 발생 기준 |
| `revisit_rate_d7` | revisit_d7 / user_count × 100 | |
| `reentry_d7` | 발표일 기준 7일 내 재응모(타입A or B) 유저 수 | |
| `reentry_rate_d7` | reentry_d7 / user_count × 100 | |
| `revisit_d30` | 발표일 기준 30일 내 재방문 유저 수 | |
| `reentry_d30` | 발표일 기준 30일 내 재응모 유저 수 | |

### 출력 형식

```
announce_date, result_type, user_count,
revisit_d7, revisit_rate_d7(%), reentry_d7, reentry_rate_d7(%),
revisit_d30, reentry_d30
```

### 어린이날 코호트 별도 요청

```
대상 정의: 2026-04-20(월) ~ 2026-04-26(일) 응모기간 동안
           5/5~5/10 경기에 응모권을 사용한 유저
발표일: 2026-04-27(월) 오전 10시
분리: 당첨자 / 미당첨자
추적 기간: 발표일 이후 7일(~5/4) / 14일(~5/11) / 30일(~5/27)
지표: 재방문율, 재응모율, 재응모 시 사용한 응모권 수
```

## 요청 4. 앱 설치 여부 × 응모 행동 비교

### 목적
앱 미설치 유저(결과 확인 불가)가 응모 참여에서 더 빨리 이탈하는지 확인

### 유저 분류 정의

| 분류명 | 정의 |
|---|---|
| `web_only` | 가입 후 2026-06-03 기준 앱 미설치 상태 유저 |
| `app_installed` | 웹 가입 후 앱 설치 완료 이력 있는 유저 |

> 앱 최초 설치일 기준으로 분류. 설치 후 삭제해도 `app_installed`로 유지

### 지표 정의

| 지표명 | 정의 |
|---|---|
| `user_count` | 분류별 유저 수 |
| `ever_entry_rate` | 가입 후 1회 이상 응모한 유저 비율 |
| `total_entry_avg` | 1인당 누적 응모 횟수 평균 (타입A+B 합산) |
| `type_b_rate` | 타입B(광고 응모) 사용 경험 유저 비율 |
| `days_to_install_median` | 웹 가입일 → 앱 설치일 경과 일수 (중앙값) |
| `entry_before_install` | 앱 설치 이전 응모 경험이 있는 유저 수 |
| `entry_rate_before` | 앱 설치 이전 응모율 |
| `entry_rate_after` | 앱 설치 이후 응모율 (설치 후 30일 기준) |

### 출력 형식

```
user_type, user_count, ever_entry_rate(%), total_entry_avg,
type_b_rate(%), days_to_install_median, entry_before_install,
entry_rate_before(%), entry_rate_after(%)
```

## 요청 5. 응모권 발급 사유 분류 유저별 행동 비교

### 목적
응모권을 어떻게 획득한 유저가 응모를 더 많이 하는지 확인

### 유저 분류 정의

| 분류명 | 정의 |
|---|---|
| `pred_reward_only` | 예측 보상으로만 응모권을 획득한 유저 |
| `event_reward_only` | 이벤트 보상으로만 응모권을 획득한 유저 |
| `mixed` | 두 가지 이상 경로로 획득한 유저 |

### 지표 정의

| 지표명 | 정의 |
|---|---|
| `user_count` | 분류별 유저 수 |
| `avg_issued` | 1인당 평균 총 발급 응모권 수 |
| `avg_used` | 1인당 평균 총 소진 응모권 수 |
| `use_rate` | avg_used / avg_issued × 100 (응모권 소진율) |
| `avg_entry_count` | 1인당 평균 총 응모 횟수 (타입A+B) |
| `type_b_rate` | 타입B 사용 경험 비율 |

### 출력 형식

```
user_type, user_count, avg_issued, avg_used, use_rate(%),
avg_entry_count, type_b_rate(%)
```

## 요청 6. 가입 코호트별 리텐션

### 목적
어느 시기 가입자가 언제 이탈하는지, 채널별 품질 차이

### 지표 정의

| 지표명 | 정의 |
|---|---|
| `cohort_week` | 가입 주차 (형식: 2026-W13) |
| `channel` | 유입 채널 (에어브릿지 channel 원본값 사용) |
| `cohort_size` | 해당 주·채널 가입자 수 |
| `retention_d1` | 가입 익일 재방문율 = 가입 다음날 활동 유저 / cohort_size |
| `retention_d7` | 가입 후 7일 내 재방문율 |
| `retention_d14` | 가입 후 14일 내 재방문율 |
| `retention_d30` | 가입 후 30일 내 재방문율 |
| `entry_d7` | 가입 후 7일 내 응모 참여율 |
| `entry_d30` | 가입 후 30일 내 응모 참여율 |

> **재방문 정의**: 가입일과 다른 날짜에 예측·응모·퀴즈·광고시청 이벤트 발생  
> **탈퇴 유저 포함**: 이탈 원인 분석이 목적이므로 탈퇴 유저도 코호트에 포함

### 출력 형식

```
cohort_week, channel, cohort_size,
retention_d1(%), retention_d7(%), retention_d14(%), retention_d30(%),
entry_d7(%), entry_d30(%)
```

### 기간
2026-W13(3/23~) ~ 2026-W22(5/25~)
