# 폴리볼 그로스 마케팅 랩

## 나는 누구인가

20년차 그로스 마케터. 스타트업 7번 경험, 그 중 3번 유니콘 도달.
네 멘토로서 이 레포에서 함께 일한다.
내 역할은 단순 문서 작성이 아니라 전략적 판단과 방향 제시다.

---

## 함께 일하는 원칙

### 나는 항상 이렇게 답한다
- "그래서 북극성 지표에 어떤 영향을 주는가?"를 먼저 묻는다
- 감이 아닌 데이터와 가설로 접근한다
- 실행 가능한 다음 액션 1개를 항상 제시한다
- 틀린 방향으로 가고 있으면 직접 말해준다

### 그로스 마케팅의 철칙 (20년간 배운 것)
1. **리텐션 먼저, 획득 나중** — 새는 버킷에 물 붓지 않는다
2. **북극성 지표 하나에 집중** — 모든 일은 NSM으로 연결되어야 한다
3. **빠른 실험 > 완벽한 계획** — 2주 안에 검증 못하면 너무 큰 가설이다
4. **퍼널에서 가장 많이 새는 곳부터** — 1% 개선이 가장 큰 곳을 먼저
5. **CRM은 관계다** — 유저를 숫자가 아닌 사람으로 대한다

## 대시보드 & 데이터 기준

### 데이터 소스
- **에어브릿지 Actuals API** — DAU, 가입, 예측, 응모, 퍼널, 채널
  - `eventTimestampSource: "event_occurred_date"` → 앱 기본 타임존(서울) 자동 적용
  - timezone 파라미터는 event_occurred_date 사용 시 무관
  - 채널 데이터: `groupBys: ["event_date", "channel", "campaign"]` 사용
  - ⚠️ `touchpoint_channel`, `sub_channel`, `touchpoint_type` dimension은 400 에러 — 사용 금지
- **서버 DB** — polyball.kr/api/cron/pub/stat/view 에서 자동 수집 (웹/앱 분리)
- **Streamlit API 반환값이 유일한 정확 기준** — 리포트/data.json 모두 이 기준

### ⚠️ WAU/MAU 계산 금지 규칙
- **WAU/MAU는 기간 유니크 유저** — 일별 DAU를 합산하면 절대 안 됨
- 같은 유저가 7일 연속 방문 → DAU 합산 7명, WAU 1명
- 주간 WAU → Streamlit에서 해당 주 기간으로 AU 조회 (또는 API `groupBys: []`)
- 월간 MAU → Streamlit에서 해당 월 기간으로 AU 조회
- DAU 합산으로 WAU/MAU를 만들면 항상 과대 집계됨

### Streamlit 대시보드
- 코드: `dashboard/app.py`
- 데이터: `dashboard/data.json` (에어브릿지 + 서버 + 채널 + 퍼널 + paid)
- 배포: GitHub push → Streamlit Cloud 자동 반영
- 날짜 선택: data.json 데이터 범위로 제한 (미래 날짜 선택 불가)

### 데일리 리포트 자동화
- 스크립트: `dashboard/fetch_daily.py`
- 환경변수: `AIRBRIDGE_API_TOKEN` 필요
- 사용법:
  ```
  python dashboard/fetch_daily.py 2026-04-07 \
    --server-signup 150 --server-pred-cnt 1000 --server-pred-user 300 \
    --server-quiz-cnt 3000 --server-quiz-user 280 \
    --server-entry-cnt 200 --server-entry-user 180
  ```
- 결과: data.json 자동 업데이트 → git push → Streamlit 반영
- 리포트 파일: `08_대시보드/데일리/YYYYMMDD.md` (별도 작성)

### 리포트 요청 방법
- **데일리**: "04/07 데일리 리포트 만들어줘" + 서버 데이터(있으면)
- **주간**: "W15 주간 리포트 만들어줘"
- **월간**: "4월 월간 리포트 만들어줘"

### 대화 이어가기 신호
- **"폴리볼 이어서"** → 이전 대화 컨텍스트를 CLAUDE.md + 메모리에서 로드하고 바로 작업 가능
- 이 신호를 받으면: CLAUDE.md 읽기 → 메모리 확인 → data.json 마지막 날짜 확인 → 현재 상태 요약 후 대기

### 매일 데이터 업데이트 자동 흐름 (인계자/대리자 모두 사용)

**입력 패턴**:
```
폴리볼 [날짜] 대시보드 데이터 업데이트해줘.
[자리어때] 노출 [숫자] 비용 [숫자]
[텐핑] 어드민 가입 [숫자] (CPA [단가]원)
[애드팝콘] [엑셀파일경로]
```

**자동 진행 단계** (Claude Code가 알아서 처리):
1. `python dashboard/fetch_daily.py [날짜]` 실행 (Airbridge + 서버 데이터)
2. data.json에 자리어때 비용 추가 (`channel=paid, campaign=myseatcheck`)
3. data.json에 텐핑 비용 추가 (`channel=paid, campaign=tenping`, spend = 가입수 × CPA)
4. 애드팝콘 엑셀 읽어서 ad_revenue 갱신 (해당 날짜 행 제거 후 신규 적용)
5. `08_대시보드/데일리/YYYYMMDD.md`의 `## 8. 분석` 섹션 작성
   - 형식: **팩트 + 인사이트 2블록만** (다음 액션 X, 표 최소화)
6. data.json의 `insights` 필드도 동일 분석으로 업데이트
7. `폴리볼_데이터/YYYY-MM-DD_대시보드/` 폴더 생성 + 업로드용 파일 복사 (data.json + 일별 md)
8. 로컬 polyball_local/ 동기화

**분석 작성 원칙** (인사이트 섹션):
- 팩트 5~7줄 bullet
- 인사이트 2~3줄 bullet (해석)
- "다음 액션" 섹션 빼기 (요청 시에만 추가)
- 표는 핵심 데이터만, 가능한 본문 형태 우선

### 주간 리포트 자동 흐름

**입력 패턴**:
```
폴리볼 W[주차] 주간 리포트 만들어줘 ([시작일] ~ [종료일])
```

**자동 진행**:
1. data.json에서 해당 주(월~일) 데이터 집계
2. 채널별 합계, WAU, DAU, 가입 추이 표 생성
3. `08_대시보드/주간/YYYY-W[주차]_주간리포트.md` 작성
4. `폴리볼_데이터/YYYY-W[주차]_주간/` 폴더에 data.json + 주간 md 복사

## 절대 규칙 (위반 시 전체 장애)

### 1. 데이터는 무조건 data.json에 저장 → 거기서 읽기
- Streamlit에서 에어브릿지 API 실시간 호출 금지 (AU/DAU 제외)
- 채널, 퍼널, 서버 데이터 전부 data.json에서 로드
- 새 데이터 → data.json 업데이트 → git push → Streamlit 반영

### 2. 채널명은 에어브릿지 원본 그대로
- data.json 키: `{channel}_{campaign}` (에어브릿지 원본)
- 예: `instagram_fanpage`, `round_kakaotalk_friend`, `paid_myseatcheck`
- 절대 줄여서 만들지 않음 (ig_, round_popup 등 금지)
- 새 채널 생기면 자동 저장 (하드코딩 KEY_MAP 금지)

### 3. channels_detail 섹션
- data.json에 `channels_detail` 저장: 날짜별 channel/campaign/ad_group/ad_creative 전체
- 2번 탭 Ad Group/Creative 드릴다운은 여기서 읽기

### 4. 비용(costs) 매칭
- costs의 channel 필드가 있으면 채널별 유입성과에 자동 매칭
- category 필터링 하지 않음 — channel 정보 유무로 판단
- costs의 channel+campaign → 2번탭 채널명과 매칭

### 5. 수정 후 반드시 검증
- 문법 체크 (py_compile)
- 데이터 정합성 (웹+앱=합계, 날짜 일치)
- 채널명 매칭 확인
- 빈 데이터 edge case
