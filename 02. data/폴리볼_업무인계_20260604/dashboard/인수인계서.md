## 3. 매일 작업 (Claude Code에 이렇게 지시)

### Step 1. 데일리 리포트 생성 — 명령 한 줄
애드팝콘 xlsx를 `C:\Users\rest\Downloads\YYYYMMDD_YYYYMMDD.xlsx` 에 받아둔 뒤:

```
6/3 데일리 리포트 만들어줘.
```

비용 데이터 있으면 같이:
```
6/3 데일리 리포트 만들어줘.
[자리어때] 노출 50000 비용 166666
[텐핑] 어드민 가입 50 (CPA 500원)
```

→ Claude가 알아서:
1. 에어브릿지 + polyball.kr 서버 데이터 수집 → `data.json` 갱신
2. Downloads의 애드팝콘 xlsx(종료일 최신 자동선택) → 광고매출 갱신 (환율 1400 고정)
3. `폴리볼_데이터\2026-06-03_대시보드\` 에 `20260603.md` + `data.json` 생성 (**전주 동요일 비교 표 포함**)
4. md에 팩트 + 인사이트 + 다음 액션 작성

### Step 2. 회사 대시보드 업로드 (사람이)
1. 회사 대시보드 마스터 로그인
2. `폴리볼_데이터\2026-06-03_대시보드\` 의 **`data.json` + `20260603.md`** 두 파일 업로드
3. 새로고침 → 분석 정상 표시 확인

### 며칠치 한 번에
```
6/2, 6/3 데일리 리포트 둘 다 만들어줘.
```

### 광고매출 익일 재정산
MAX는 당일 미정산 → 당일 리포트는 매출 0/요청만. **다음날 새 xlsx 받고:**
```
6/3 광고매출 다시 반영해줘. (어제 미정산분 확정됐어)
```

## 4. 주간 / 특수 분석

**주간 리포트** (월요일):
```

**특수 분석** (예: 미니게임):

```
→ `08_대시보드\미니게임_분석_YYYYMMDD.html` (다크테마, 차트, 자체완결 HTML).

**즉석 데이터 질문** — 그냥 물어보면 Claude가 에어브릿지에서 바로 뽑음:
```

6월 1주 WAU 알려줘.   ← (WAU/MAU는 기간 유니크로 뽑음, DAU 합산 안 함)
```

## 5. 절대 규칙 (Claude도 지키지만, 사람이 결과 검증할 때 기준)

1. **데이터는 무조건 `data.json` 저장 → 거기서 읽기.** 새 데이터 → data.json 갱신 → 업로드.
2. **WAU/MAU = 기간 유니크. 일별 DAU 합산 금지** (항상 과대집계).
3. **채널명은 에어브릿지 원본 그대로** (`instagram_fanpage` 등). 줄이지 않음.
4. **환율 USD×1400 고정.**
5. **데일리 리포트엔 "전주 동요일 비교" 표가 반드시 있어야 함** (예: 05-27 수 vs 06-03 수). 없으면 Claude에게 다시 만들라고 지시.
6. **요일 지표 낮다고 "결손" 단정 금지** — 전주 동요일과 비교해 요일 계절성인지 먼저 확인.

**검증 체크리스트** (업로드 전 눈으로):
- [ ] 날짜 맞나 / 웹+앱 합 = 합계 맞나
- [ ] 전주 동요일 비교 표 있나
- [ ] 광고매출 0이면 "미정산" 표기됐나 (익일 재반영 예약)

## 6. 내부 동작 (Claude가 막히거나 사람이 직접 돌릴 때만)

Claude는 아래를 자동 실행한다. 수동 재현이 필요하면:

```cmd
cd C:\Users\rest\my-growth-lab\dashboard

# 1) 에어브릿지+서버 → data.json  (※ argv 무시됨, p.main()로 호출)
python -c "import daily_pipeline as p; p.main('2026-06-03')"

# 2) 애드팝콘 xlsx → ad_revenue (Downloads의 YYYYMMDD_YYYYMMDD.xlsx 자동선택, FX=1400)
python _ingest_adpopcorn.py

# 3) 폴리볼_데이터 데일리(전주 동요일 비교 포함) — daily_pipeline 아님!
python build_daily_polyball.py 2026-06-03

# 특수: 미니게임 HTML
python build_minigame_report.py
```

**에어브릿지 직접 뽑기** (Claude가 즉석 분석할 때 쓰는 두 API):
- **Actuals API** = 집계값. metric 형식 `{web|app}_custom_users_{이벤트}`. `daily_pipeline.ab_request(payload)` → `ab_parse_rows()`. 두 계정(polyball/polyballkr) 자동 합산. 채널은 `groupBys:["event_date","channel","campaign"]`. ⚠ `touchpoint_*`/`sub_channel` = 400 에러 금지.
- **Log-export API** = 원시 이벤트(라벨/액션). 게임별 분리 등 세부 분석용. 비동기(요청→폴링 최대 5분). 템플릿 = `build_minigame_report.py`.
- 미니게임 라벨: `bbada`=야구빠따, `4taja`=4번타자, `fishing`=낚시하기. 미니게임/광고 커스텀 이벤트는 **5/19부터** 존재, 앱 전용 집계.
- 이벤트명 모를 때: `python _list_events.py`

**자주 하는 실수**:
- `python daily_pipeline.py 2026-06-03` → argv 무시(어제로 감). `p.main('2026-06-03')` 사용.
- `data.json["daily"]` 는 **list** (`[{"date":...}]`), dict 아님.

## 7. 트러블슈팅

| 증상 | 해결 |
|---|---|
| Claude가 멈춤 | 같은 명령 다시 입력 |
| `ModuleNotFoundError` | `pip install -r requirements.txt` + `openpyxl toml` |
| 에어브릿지 401/403 | secrets.toml 토큰 만료 → 콘솔 재발급 |
| 서버 데이터 0 | 집계 지연. 30분 후 재시도, 안 되면 개발팀 |
| 광고매출 0 / 노출만 | MAX 당일 미정산 → 익일 xlsx로 재반영 |
| 애드팝콘 xlsx 못 찾음 | 파일명 `YYYYMMDD_YYYYMMDD.xlsx`, Downloads 위치 확인 |
| 전주 동요일 표 없음 | Claude에게 "build_daily_polyball.py로 다시 만들어" |
