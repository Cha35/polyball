PK-01 Pick Main (픽 메인) 오늘의 경기 예측
Description
OVERVIEW 화면 개요
오늘(당일 기준) 진행되는 KBO 경기의 승부 및 경기 흐름을 예측하는 메인 화면
메인마켓(승리팀 예측)과 서브마켓(경기 흐름 예측)을 각각 별도 카드로 제공
마켓은 매일 00:00에 오픈되며, 경기 시작 시간까지 예측 가능
예측 마감 후 미선택 마켓은 선택지 영역이 제거되고, 선택 완료 마켓은 버튼 유지
적중 시 응모권 획득 (기본 1개, x2배 찬스 적용 시 2개)
COMPONENT 주요 UI 구성
GNB: PICK 로고 (좌) / 응모권 아이콘 + 실시간 보유 수 (중) / 알림 아이콘 (우)
응모권 아이콘 클릭 시 응모 페이지로 이동 (GNB 응모 탭과 동일)
알림 아이콘 클릭 시 알림함으로 이동
앱 설치 배너: "PICK 앱 설치하고 푸시 알림 받기" + 설치하기 버튼 + X 닫기
설치하기 클릭 시 OS별 앱스토어(플레이스토어/앱스토어)로 이동
X 클릭 시 배너 닫기 (localStorage.appBannerDismissed = 'true')
지난 예측 결과 배너: 🎯 아이콘 + "지난 예측 결과 확인하기 / 나의 승부픽 적중률 보기" → PK-13 이동
섹션 헤더: "오늘의 승부PIKK!" + 잔여 마켓 수 표기 (예: 20 예측 참여 가능)
서브 텍스트: "예정된 경기를 예측해 보세요."
경기 카드 그룹: 응원팀 배지 + 메인마켓 카드 + 서브마켓 카드들
LOGIC 핵심 로직
경기 데이터 로드: 당일 경기 데이터를 날짜 기준으로 로드
예측 상태 관리:
mainPicks: Record<gameId, {team: 'home'|'away'}>
subPicks: Record<gameId, Record<situationId, 'optA'|'optB'>>
localStorage.pick_participated, localStorage.submarket_picks에 저장
경기 상태 판단:
open: 00:00 ~ 경기 시작 시간 (예측 가능)
closed: 경기 시작 시간 이후 (예측 마감)
마감 시 UI 처리: 미선택 마켓은 선택지 버튼 영역 제거, 선택 완료 마켓은 선택 완료 버튼 유지
정렬 우선순위: 1) 응원팀 경기, 2) 미완료 경기, 3) 경기 시간순
INTERACTION 예측 인터랙션
예측 버튼 클릭: 확정 모달 표시
모달 타이틀: "[팀명] 승리로 예측을 확정하시겠어요?"
모달 서브: "한 번 선택하면 변경할 수 없어요."
"확정" (검정 버튼) / "취소" (화이트 버튼)
예측 후 변경: 불가 (선택 완료 버튼으로 전환, 비활성)
x2배 찬스: 선택률 30% 이하일 때 "30%↓ 적중 시 2배🔥" 툴팁 표시
선택 전/후 모두 툴팁 유지, 해당 선택지 위에 표시
CARD 메인마켓 카드
배경: 다크 그라데이션 + 번개/조명 효과 배경 (구장 이미지 희미하게 적용 예정)
뱃지: 🏆 승부예측
매치업: 원정팀명 (좌) / 구장명+날짜시간 (중, 블루 텍스트) / 홈팀명+H뱃지 (우)
프로그레스바: 원정팀/홈팀 선택률 시각화 (오렌지/다크)
[선택 전 - open]
원정팀 버튼 (오렌지 배경) / 홈팀 버튼 (검정 배경)
선택률 30% 이하 선택지: "30%↓ 적중 시 2배🔥" 툴팁 표시
[선택 후 - open]
"[팀명] 승리 선택 완료" 단일 버튼 (회색, 비활성)
x2배 툴팁 유지
[마감 - 미선택]
선택지 버튼 영역 제거, 하단 "예측종료" 표시
[마감 - 선택 완료]
선택 완료 버튼 유지 (회색, 비활성), 하단 "예측종료" 표시
하단 정보: 👥 N명 참여 / ⏱ 종료까지 HH:MM:SS 또는 예측종료 / 🎫 +1
CARD 서브마켓 카드
배경: 화이트, 라운드
뱃지: Q (질문 표시) + 질문 텍스트
프로그레스바: optA/optB 선택률 시각화 (블루/회색)
[선택 전 - open]
optA 버튼 / optB 버튼 (각각 선택 가능)
[선택 후 - open]
"[선택지명] 선택 완료" 단일 버튼 (회색, 비활성)
[마감 - 미선택]
선택지 버튼 영역 제거, 하단 "예측종료" 표시
[마감 - 선택 완료]
선택 완료 버튼 유지 (회색, 비활성), 하단 "예측종료" 표시
하단 정보: 👥 N명 참여 / ⏱ 종료까지 HH:MM:SS 또는 예측종료 / 🎫 +1
POLICY 정책 및 규칙
마켓 오픈: 매일 00:00 당일 경기 마켓 오픈
단, 매주 월요일은 경기 없는 날 → 화요일 경기 마켓이 월요일 00:00에 오픈
예측 가능 시간: 00:00 ~ 경기 시작 시간
예측 수정: 불가 (최초 선택 후 변경 불가)
메인마켓: 해당 경기의 원정팀 vs 홈팀 승리팀 예측, 경기 전체 마켓의 타이틀 역할
서브마켓: 해당 경기의 부가 마켓, 2개의 선택지로 구성
메인/서브 독립성: 메인 예측 없이도 서브 예측 가능
보상: 적중 시 응모권 1개, x2배 찬스 적중 시 2개
x2배 찬스: 마감 시점 선택률 30% 이하 → 적중 시 2배 지급
특수 경기 결과 처리: 무승부 / 우천취소 등 원정팀·홈팀 선택지에 해당하지 않는 결과는 전원 적중 처리
POLICY 출석 보상
지급 시점: 당일 최초 접속 시 자동 지급 · 상단 토스트 메시지로 안내
일반 요일 (화,수,목,금,일): 응모권 +1장
보너스 요일 (월,토): 응모권 +2장 (기본 1장 + 보너스 1장 추가 지급) — 월/토 접속률 제고 목적
연속 출석: 무제한 누적 — 연속 출석일수별 추가 보상은 추후 설계 예정
재지급 없음: 재진입/새로고침 시 중복 지급 없음

PK-13 내 예측 히스토리 나의 예측 결과 및 응모권 적중 내역 확인
Description
OVERVIEW 화면 개요
나의 전체 예측 내역과 적중/실패 결과를 확인하는 화면
메인마켓과 서브마켓 예측이 날짜별 리스트로 통합 표시
적중 여부와 획득 응모권 수를 각 항목별로 표시
"더보기" 버튼으로 추가 내역 로드
COMPONENT UI 구성 요소
헤더: ← 백 버튼 + "내 예측 히스토리"
유저 카드: 아바타 + 닉네임 + 응원팀 뱃지(MY [팀명]) + 통계 3개
통계: 총 예측 수 / 예측 적중 / 예측 적중률
섹션 타이틀: "전체 예측 내역"
예측 리스트: 날짜 / 마켓명 / 내 선택+결과 / 적중 또는 실패 뱃지
적중: "적중 +N" 뱃지 (블루)
실패: "실패" 뱃지 (회색)
더보기 버튼: 하단 "더보기 ∨" 클릭 시 추가 내역 로드
LOGIC 데이터 변환 로직
예측 내역을 날짜 역순(최신순)으로 리스트 표시
각 항목: 날짜 / 경기명+마켓명 / 내 선택 및 결과 / 적중 여부 + 획득 응모권
"더보기" 클릭 시 추가 내역 페이지네이션 로드
DATA 데이터 구조
PredictionRecord: id, date, awayTeam, homeTeam, myPrediction, gameResult, subMarketPicks, subMarketResults, situations
PredictionCard: id, date, type('main'|'sub'), gameInfo, marketName, myChoice, result, isCorrect, ticketsEarned
EDGE CASE 예외 케이스
경기 취소: gameResult === 'canceled' → 응모권 반환 미정의
무승부: gameResult === 'draw' → 처리 방법 미정의
