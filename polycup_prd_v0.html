폴리컵 리그 기능정의서

버전 v2.0 · 2026-06-29 · 폴리볼 2.0 / Wallet 사업실 · 공유용 요약본
v2.0 핵심 변경 — 폴리컵을 ‘상시 통합 랭킹’이 아니라 시즌·이벤트 단위로 선별 개설하는 리그형 이벤트로 재정의한다. 랭킹은 리그 랭킹(보상)과 누적 랭킹(명예·기록)으로 이원화하며, 동시 다중 리그 운영을 전제로 한다.
확정 임시(출시 후 정교화) 미결정 신규

1. 제품 정의

폴리컵 리그가 무엇이고, 무엇이 아닌가

한 줄 정의

폴리컵 리그는 일정·정답·트래픽이 명확한 시즌성 예측 콘텐츠에 한해 선별적으로 여는 리그형 예측 이벤트다. 모든 예측에 자동으로 붙는 상시 랭킹 시스템이 아니다.

구조 원칙 확정

  • 아니다 — “폴리컵 안에 모든 예측이 있다” (상시 통합 모델 폐기)
  • 맞다 — “예측 서비스 안에 여러 카테고리가 있고, 그중 특정 이벤트에 폴리컵 리그가 열린다”

BM 연결

참여 모멘텀이 큰 이벤트에 경쟁·보상을 집중 → 픽/응모 빈도 상승 → 광고 노출 기회 증가. 리그 사이 공백기는 누적 랭킹(명예)으로 상시 참여 동기를 유지한다.

2. 리그 개설 대상 기준

아무 카테고리에나 리그를 붙이지 않는다

4개 조건 — 모두 충족 시에만 개설 확정

#조건
1유저 참여·트래픽이 충분히 기대되는 주제
2일정이 명확한 주제
3정답·결과 판정 기준이 명확한 주제
4일정 기간 예측 콘텐츠를 꾸준히 공급할 수 있는 주제

✓ 대상 (폴리컵)

KBO · EPL · 챔피언스리그 · KBL · 월드컵 · 아시안게임 등 시즌성 스포츠/대회

✕ 제외 (→ 폴리픽)

연애·예능·사연·여론 등 정답 없는 의견형. 폴리컵 리그 집계 대상 아님

3. 동시 다중 리그 운영

여러 리그가 동시에 열리는 것을 전제로 설계한다

KBO 리그가 도는 중에 월드컵·MSI 리그가 동시에 열릴 수 있다. 따라서 다음을 설계 대상으로 한다.

영역결정해야 할 것
운영동시 N개 리그의 개설·정산·보상 병렬 관리 방안
진입점여러 리그가 열렸을 때 유저가 각 리그로 들어가는 경로
UI 구조‘진행 중 리그’ 목록 화면 + 카테고리 필터([전체]+대분류 동적 탭)
상세 페이지개별 리그의 기간·상태·랭킹·마켓·보상 조회 구조
주의 — 이전에 폐기했던 ‘리그 선택’ 화면이 다중 리그 전제에서는 다시 필요해진다. 단, “유저가 참여를 고르는” 화면이 아니라 “열린 리그를 탐색·진입하는” 목록으로서 필요하다.

4. 랭킹 이원화

보상은 리그 단위, 명예는 누적 단위로 분리한다

구분리그 랭킹누적 랭킹
목적경쟁 · 보상명예 · 기록
범위개별 리그 기간 내 성과카테고리 무관 전체 누적
보상지급 기준 (O)보상 아님 (X)
표현리그 랭킹 보드개인 트랙레코드 (전문 분야)

누적을 보상 기준으로 쓰지 않는 이유: 참여량 많은 유저가 과도하게 유리해지고, ‘적중률 vs 참여량’ 우열 기준이 모호해 공정성 논란이 생긴다.

통합 ‘폴리컵 리그’(가칭) 미결정
카테고리를 아우르는 전체 통합 랭킹을 둘지 검토 중. 방향은 보상 없는 명예·누적 레이어로 존치(주인공은 개별 이벤트 리그). 노출 위계 미결정 — (가) 별도 랭킹 화면 / (나) 마이 트랙레코드로만 표현.

5. 점수 산정 확정

리그 내 점수 규칙 (기능명세서 기준 유지)

선택률 가중 점수

적중 조건점수
적중 기본100
선택률 30~21% 적중150
선택률 20~11% 적중180
선택률 10~1% 적중200

선택률은 예측 마감 시점에 고정. 소수가 맞힌 예측(역배)일수록 고점.

랭킹 산출 · 동점 처리

  • 리그 랭킹은 리그 기간 내 누적 점수 내림차순
  • 동점 처리: ①적중 수 → ②userTime → ③hash(리그+유저), 공동 순위 미허용
  • userTime = (마감−제출) 시각 합산. 클수록 = 일찍 예측 → 상위. 내부 로직(유저 비노출)

어뷰징 가드레일 임시

마켓 참여자 N명 미만이면 선택률 가중 제외(기본 100점만 부여). 소수 담합에 의한 고점 방지. N은 운영자 설정.

6. 순위 표시 — 예상 / 확정 확정

결과 확정 전후를 구분해 보여준다

확정 순위

결과 확정된 적중분만 반영. 보상 기준. (예: 142위 · 1,180점)

예상 순위

“진행 중 예측이 모두 적중하면 최대 +540점 → 88위”. 픽 직후·랭킹 보드 노출.

  • 예상치에는 “모두 적중 시 / 최대” 조건 표기 필수 (단일 숫자 노출 금지 → 급락 클레임 방지)
  • 예상 등수는 “본인 예상 점수를 현재 순위표에 끼워넣어 위치 산출” → 전체 재정렬 없이 구현, 부하 낮음
  • 부하 발생 시 폴백: 점수만 표시 (등수 생략)

7. 기간 · 참여 · 보상

리그 기간 / 리셋 재검토

리그가 이벤트 단위이므로 리셋은 리그 기간 단위로 정의(이벤트마다 상이). 이전 ‘주간 일괄 리셋’은 폐기. 점수 귀속 = 결과 확정 시점이 속한 기간.

최소 참여 / 랭킹 노출 재정의

  • 랭킹 노출: 조건 없음(픽 1회부터 표시)
  • 보상 대상: 리그 기간 내 최소 유효 참여 기준 충족 시 (수치는 리그별 운영자 설정, 초기 기준값 검증 후 확정)
  • 미달자: 랭킹엔 노출 + “보상까지 N회 더” 진행 표시 → 추가 픽 유도

보상 기준 확정

보상은 리그가 열린 기간 내 성과 기준으로 지급. 전체 누적 랭킹 기반 보상은 폐기. KBO는 보상 재화가 전용(직관티켓→경품 응모)이며 별도 고지.

8. 노출 · 안전 정책 확정

  • 닉네임 마스킹 — 랭킹 보드 전원 마스킹(홍*동). 본인 행은 강조 + “나” 라벨. 닉네임 자유설정 가능하나 초기 디폴트값 자동지정
  • 공개 동의 — 가입 약관에 “랭킹·닉네임 공개” 포함. 필수항목 개정이므로 기존 회원에 개정 고지 + 미동의 시 탈퇴 권장
  • 신고 / 차단 — 신고 누적·제재 유저는 닉네임 가림 또는 랭킹 제외
  • 비로그인 — 리그 홈·랭킹 보드·보상 안내 열람 가능. 개인화(내 순위·내 보상)는 로그인 후, 원래 맥락 복귀
  • 빈 상태 — TOP 랭킹은 항상 노출. 내 순위 자리는 placeholder+CTA(픽0회=첫픽 / 비로그인=로그인). 동일 템플릿, CTA만 분기
