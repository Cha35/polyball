경품 어드민 — 응모 회차 구조 개편 (초안)

실제 구현된 Wallet Admin 화면 검토 결과 확인된 "경품 = 응모 1:1" 구조 문제를 해결한다. 경품 마스터와 응모(회차)를 분리하고, 회차 안에서 경품 여러 개 · 경품별 당첨 인원/수량을 지정하는 지점을 신설한다.
기능 ID ADM-PZ 버전 v2.1 (초안) 플랫폼 BO(Wallet Admin) 기반 ADM-PZ v1.0 실 구현 화면 리뷰 작성일 2026-07-06
초안 범위 안내 — 이 문서는 실 구현 화면(경품 상품 관리)에서 발견된 구조 문제(경품=응모 1:1) · 당첨 인원/수량 필드 부재를 해결하는 데 범위를 한정한다. 확정보상·최다 응모자·본인인증 응모 게이트·제세공과금 자동 트리거 등 기존 v2.0 기획안의 추가 정책은 이번 구조 개편이 확정된 후 별도 반영한다(§TODO 참조). 필드명·화면 배치는 제안 값이며 개발 검토 후 확정한다.
PZ-00개요 · AS-IS 문제 정리

경품 어드민은 인앱 마이 > 내 지갑 > 보유 경품(WL)에 표시되는 경품 데이터를 생성·운영하는 상위 도구다. 실 구현 화면을 리뷰한 결과, 아래 5가지 구조적 문제가 확인되어 개편이 필요하다.

AS-IS 문제 · 실 화면 근거
문제실 화면 근거영향
① 경품=응모 1:1 구조「경품 상품 목록」에 "닌텐도 스위치2"가 SEQ 1 · SEQ 3으로 중복 등록되어 있고, 각각 다른 응모 시작/종료일을 가짐. 「경품 상품 등록」 모달에 응모 시작/종료일시가 경품 필드로 박혀 있음같은 경품을 다른 기간에 재사용하려면 전체 정보를 재입력해야 함. 마스터 재사용이 무력화됨
② 당첨 인원·수량 필드 부재등록 모달·목록 어디에도 "몇 명에게 몇 개씩"을 입력·표시하는 필드가 없음재고 검증·추첨 배치를 실행할 근거 데이터 자체가 없음
③ 노출/응모 기간 미분리「경품 상품 등록」에 응모 시작/종료일시만 존재, 노출 기간 필드 없음마감 후에도 상품을 계속 열람시키는 운영이 불가능
④ 활성화가 경품 단위「경품 상품 목록」의 🚫 아이콘이 경품(행) 단위로만 존재응모 하나를 통째로 멈추려면 그 안 경품을 각각 꺼야 함
⑤ 쿠폰코드관리 위치 혼동「쿠폰 코드 관리」(제휴사 수기 쿠폰, 응모 미사용)가 「쿠칩 상품 관리」 하위에 나란히 위치운영자가 응모용 코드 관리로 오인할 위험
운영 사이클 (TO-BE)
경품 마스터 등록→ 응모 회차 등록→ 경품 추가 · 당첨설정→ 서버 배치 추첨→ 당첨자 관리→ 인앱 WL 노출
굵게 표시한 두 단계(응모 회차 등록 / 경품 추가·당첨설정)가 이번 개편으로 신설되는 지점이다. 나머지 단계는 기존 화면을 그대로 유지한다.
