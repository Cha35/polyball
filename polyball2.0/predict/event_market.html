0 개요 · 작업 범위
본 문서는 이벤트 마켓(BO 수기 등록)의 예측형·참여형 카드 프론트 UI 재구성과 보상 아이콘 딤드 처리를 정의합니다. 마켓 유형·정산 로직·카테고리 구조·재화 시스템은 기존 EM v1.9 / 어드민 v7을 그대로 승계하며 변경하지 않습니다.
개편 배경
기존 정의서는 마켓 카드를 유형(예측형/참여형/퀴즈형) 중심으로만 기술하고, 선택지 렌더 방식(이미지 유무·양자택일 강조)과 보상 표기(하단 텍스트 분산)를 명확히 분리하지 않았습니다. 본 개편은 유형(로직)과 렌더형(표현)을 분리하고, 흩어져 있던 보상 표기를 카드 우측 상단 아이콘으로 통합합니다.
작업 범위
포함 (IN)예측형·참여형 카드 렌더형 재정의(리스트형/양자택일형) · 카드 상태별 UI · 보상 아이콘 우측 상단 이관 및 딤드
제외 (OUT) — 정규 마켓한화 vs KT류 승/패·경기흐름 마켓은 API 자동생성 마켓으로 서버에서 데이터·정산 픽스. 프론트 재구성은 개발팀 이관. 본 문서 범위 아님
제외 (OUT) — 퀴즈형이번 개편 보류 · 현행 유지 · 추후 별도 보강
DB / 스키마표시형식 체크박스 필드 1개 신규 추가 외 변경 없음 (§5 참조)
정산 · 보상 지급 로직기존 EM v1.9 승계 · 변경 없음 (지급 시점·재화·정답 판정 동일)
1 유형 · 렌더형 구조
핵심 원칙 — 유형은 로직(정답 유무·정산·보상 시점)을 결정하고, 렌더형은 표현(선택지 배치·수치 노출)을 결정합니다. 두 축은 독립이며, 렌더형은 유형에 따라 사용 가능 범위가 제한됩니다.
1축 — 유형 (로직, 기존 승계)
유형정답보상 지급 시점정산노출 탭
🎯 예측형 있음 (마감 후 어드민 확정) 참여보상=선택 즉시 / 정답보상=정산 시 수동 스페셜 예측
✅ 참여형 없음 참여보상=선택 즉시 마감 시 자동 종료 투표
2축 — 렌더형 (표현, 개편 대상)
렌더형배치선택지 개수선택률 노출사용 가능 유형
리스트형 선택지 세로 나열 (텍스트 + 선택 시 썸네일) 2~10개 선택/참여 후 우측 % 예측형 · 참여형
양자택일형 선택지 2개 좌우 반반 · 선택률 대비 강조 2개 고정 진입 즉시 노출 (여론 강조) 참여형 전용
양자택일형이 참여형 전용인 이유 — 양자택일형은 진입 즉시 양쪽 선택률을 대비 노출하는 표현입니다. 정답이 있는 예측형에 적용하면 마감 전 선택률이 정답 힌트가 되고 예측의 긴장감을 해칩니다. 정답이 없는 참여형은 실시간 여론 자체가 콘텐츠 가치이므로 양자택일 강조가 자연스럽습니다.
렌더형 결정 규칙
리스트형 (기본값)모든 예측형 · 양자택일 체크 해제된 참여형
양자택일형참여형 + BO 표시형식 체크박스 ON + 선택지 정확히 2개
선택지 이미지렌더형과 무관 · 선택지 데이터에 이미지 있으면 리스트형 우측 썸네일 노출, 없으면 영역 생략
예측형 체크박스양자택일 체크박스는 참여형에서만 활성화 · 예측형은 비활성(선택 불가)
※ 리스트형과 "리스트 이미지형"은 별개 렌더형이 아닙니다. 이미지는 선택지 단위 옵션 요소이며, 있으면 우측에 썸네일이 렌더되고 없으면 영역이 생략될 뿐 동일 리스트형입니다.

4 보상 아이콘 · 딤드 처리 신규
기존 카드 하단에 텍스트로 분산되어 있던 보상 표기를 카드 우측 상단 아이콘 영역으로 통합합니다. 아이콘은 "받을 수 있는 보상"을 어포던스로 노출하고, 수령 완료 시 딤드(회색)로 전환하여 획득 여부를 직관적으로 표현합니다.

체크박스 유효성 — 참여형 + 체크 ON + 선택지 정확히 2개, 세 조건을 모두 충족할 때만 양자택일형으로 렌더. 선택지가 3개 이상이면 체크가 ON이어도 리스트형으로 폴백(또는 등록 시 경고). 예측형에서는 체크박스 자체가 비활성.
6 예외 · 미결 사항
#이슈옵션 / 검토 방향
1 예측형 미적중 시 정답보상 아이콘 처리 확정 — 아이콘 제거 (받을 수 없음 확정 · 참여보상은 딤드 유지)
2 미참여 상태로 마켓 종료 시 보상 아이콘 확정 — 전체 제거 (미수령 확정, 딤드 의미와 충돌 방지)
3 복수 재화 최대 종류 수 확정 — 최대 2종 (1종이 일반, 많아야 2종). 3종 이상 운영 미사용 → 별도 축약 UI 불요
4 기존 참여완료 축소 UI(선택지 제거·단일 버튼) 확정 — 폐기 · 선택지 유지 + 아이콘 딤드로 통일
5 정규 마켓과 컴포넌트 재사용 경계 양자택일형 렌더가 정규 마켓(승/패)과 시각 유사 · 컴포넌트 공유 여부 개발팀 협의
6 퀴즈형 적용 이번 보류 · 아이콘 딤드 스펙 적용 시점 추후 결정
