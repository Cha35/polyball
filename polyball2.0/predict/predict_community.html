2-1예측 근거 코멘트
구조변경신규
예측 선택 직후 "왜 그렇게 생각하는지" 한 줄 코멘트를 유도합니다. 코멘트는 마켓 댓글로 자동 누적되어 댓글 수를 늘리고(커뮤니티 진입 유도), 자신의 의견이 걸린 마켓의 결과를 확인하러 오는 재방문을 강화합니다.

동작 정의
트리거: 예측/참여 선택 확정 직후 모달 노출
입력: 한 줄 텍스트 (선택 · 건너뛰기 가능)

저장: 작성 시 마켓 댓글로 누적 (일반 댓글과 동일 리스트)
적용 유형: 예측형 · 참여형 공통
선택지 스냅샷 뱃지
뱃지 내용"예측: {선택지명}" — 코멘트 작성자가 고른 선택지 표기
스냅샷 기준작성 시점 선택지를 고정 · 이후 유저가 예측을 변경해도 그 코멘트 뱃지는 불변
근거예측은 마감 전 변경 가능 → 실시간 선택 참조 시 코멘트-선택 불일치 발생. 작성 당시 생각의 기록으로 고정
복수 코멘트마켓당 여러 번 허용 · 선택 변경 시 서로 다른 뱃지의 코멘트가 각각 누적
참여형 뱃지정답 개념 없음 → "예측:" 대신 "선택: {선택지명}" 표기 (유형에 맞는 라벨)
뱃지 노출 시점 확정
예측형·참여형 모두 코멘트 뱃지를 작성 즉시 노출합니다. 예측형이라도 마감 전 노출을 막지 않습니다.
근거: 마감 후 노출 대상은 집계 선택률(%)이지 개별 유저의 텍스트 의견이 아닙니다. 개별 코멘트 뱃지는 여론 전체를 드러내지 않아 힌트 리스크가 제한적이고, 수준 높은 코멘트가 조기에 보여야 토론이 붙어 마켓 관심도·체류가 올라갑니다(광고 노출 유리). 따라서 즉시 노출.
2-2코멘트 보상
구조변경신규
참고 정책 (본 버전 미적용) — 근거 코멘트에 재화 보상을 거는 방안은 핵심 기능에서 제외합니다. 보상을 걸면 체리피커성 저품질 댓글이 늘어 실익이 낮다는 판단입니다. 아래는 향후 도입 시 참고할 정책안으로만 남깁니다.
적용 여부본 버전 미적용 · 참고 정책으로만 보존
(참고) 지급 조건근거 코멘트 5글자 이상 작성 시 재화 1개
(참고) 일일 상한하루 5회까지 · 초과 시 작성은 되나 보상 없음
(참고) 정산 연동없음 · 정답 2배 등 정산 로직 변경 없음
설계 이력 — 초기 검토안(예측형 정답 시 정답보상 2배 / 마켓당 지급)은 재화 지출 통제 문제로 "하루 5회 상한 단일 정책"으로 축소했다가, 최종적으로 보상 자체를 본 버전에서 제외했습니다. 코멘트는 보상 유인 없이 "예측 근거 공유 → 토론·관심도 상승"이라는 자발적 참여로 운영합니다. 재화 유인은 체리피커 리스크가 커 참고 정책으로만 보존합니다.

4예측 불러오기 재정의
구조변경
존치 형태CM-02에 인용 + 링크로만 존치 — 마켓 정보(카드 요약) 인용 + 마켓 이동 네비게이션
역방향 연결해제 — 인용한 글이 마켓 댓글/피드에 역으로 걸리지 않음
인용 카드 동작탭 시 해당 마켓 상세로 이동 (링크 역할만)
기능 완전 제거본 버전 제거하지 않음 · 유지 확정 (역연결만 해제)
커뮤니티에서 마켓과 엮이는 접점은 예측 불러오기(인용 + 링크) 하나만 남습니다. 태그 기반 글쓰기(마켓 커뮤니티 탭에 역노출)는 사라지고, 인용은 마켓으로 가는 단방향 링크로만 기능합니다.

6이미지 업로드 버튼 차단
보류
보류 (본 버전 미적용) — 방통위 인증 이슈가 계도기간으로 확인되어, 이미지 업로드 버튼 차단을 보류합니다. 즉시 조치하지 않고 인증 요건이 확정되는 시점에 커뮤니티 vNext에서 정식 대응합니다.
적용 여부본 버전 미적용 · 보류 (계도기간)
(대기) 대상CM-02 글쓰기 하단 툴바 이미지 첨부 버튼 (마켓 댓글은 원래 이미지 불가)
(대기) 처리 방식차단 시 프론트 숨김만 · 백엔드/API 유지 · 축소 배포 없음
근거방통위 인증 이슈 (계도기간 → 요건 확정 시 대응)
이월커뮤니티 vNext에서 이미지 기능 정식 정의 및 차단 여부 결정
계도기간 동안은 기존 이미지 업로드·표시 정책을 그대로 유지합니다. 차단이 필요해지는 시점에 CM-02 글쓰기 버튼 숨김으로 대응하며, 그 전까지 별도 조치는 하지 않습니다.
