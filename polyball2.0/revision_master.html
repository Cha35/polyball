A-0개요 · 작업 범위
포함: 예측형·참여형 카드 렌더형 재정의 · 상태별 UI · 보상 아이콘 이관/딤드 · 댓글 수 노출
제외: 정규 마켓(API 자동생성, 개발팀 이관) 범위 밖 · 퀴즈형 보류
불변: 유형·정산·카테고리·재화 (DISPLAY_TYPE 체크박스 1개만 신규)
A-1유형 · 렌더형 구조
1축 유형(로직): 예측형(정답 O) / 참여형(정답 X) — 정산·보상 시점 결정
2축 렌더형(표현): 리스트형 / 양자택일형
예측형 → 리스트형 고정
참여형 → 리스트형 / 양자택일형 (DISPLAY_TYPE 체크 + 선택지 2개)
양자택일형 = 참여형 전용 (선택률 즉시 노출 → 예측형엔 정답 힌트 문제)
선택지 이미지 = 옵션 요소 (있으면 리스트형 썸네일, 없으면 생략 · 별도 렌더형 아님)
A-2렌더형별 카드 스펙
리스트형: 선택지 세로 나열 + 썸네일(옵션) + 선택률(선택 후)
양자택일형: 2선택지 좌우 반반 · 선택률 진입 즉시 대비 노출
A-3카드 상태 매트릭스
예측형: 미참여 → 참여·진행중 → 정산대기 → 예측성공/실패 → 미참여·종료
참여형: 미참여 → 참여완료 → 참여종료 → 미참여·종료
상태별 선택지 노출 · 선택률 노출 · 비활성 처리 규칙
A-4보상 아이콘 · 딤드 신규
신규
위치: 카드 우측 상단 아이콘 이관 (하단 텍스트 폐기)
상태 3종: 컬러(받을 수 있음) / 딤드(받았음) / 없음(받을 수 없음 확정)
예측형: 정답보상🏆 + 참여보상🎫 2개 독립 — 참여=선택 즉시 딤드, 정답=정산·적중 시 딤드
참여형: 참여보상🎫 1개 — 선택 즉시 딤드
미적중: 정답보상 제거 / 미참여 종료: 전체 제거 · 재화 최대 2종
A-5카드 하단 · 댓글 수 신규
신규
