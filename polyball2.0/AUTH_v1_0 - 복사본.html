1진입점 · 후행 동작 게이트 공유

플로우 통합 원칙 — 세 진입점 모두 요구 동의(본인인증 + 예측 통계)와 PASS가 동일하므로 게이트는 AUTH-C 하나로 공유한다. 분리 기재하지 않고, 위 표처럼 진입점·후행 동작만 구분한다.
2예측 상세 통계 잠금 상태 화면 상태
  • 미인증·미동의자가 예측 상세에 진입하면 통계 영역이 블러 처리 + 오버레이 CTA("본인 인증하고 결과를 확인해 보세요 / 확인하기")로 노출된다.
  • 이는 액션으로 뜨는 바텀시트가 아니라 상시 화면 상태다. 본인인증 + 통계 동의 완료 시 블러가 해제되고 실제 통계(성별·세대별)가 노출된다.
  • [확인하기] 탭 → AUTH-C 게이트(본인인증+통계 동의→PASS) → 성공 시 블러 해제.
3분기 로직
상태동작
본인인증 O · 통계 동의 O게이트 없이 바로 예측 참여
본인인증 O · 통계 동의 X통계 동의만 요청(바텀시트) → 동의 시 예측 참여
본인인증 X (통계 무관)본인인증 동의 + 통계 동의 함께 요청 → 동의 → PASS 실행 → 성공 시 예측 참여
4완료 처리 규칙 중요
  • 본인인증 미완료 상태에서는 두 동의(본인인증·통계)를 모두 완료로 체크해서는 안 된다.
  • 동의 + PASS 인증을 하나의 트랜잭션으로 처리 — PASS 성공 콜백 이후에만 본인인증 완료 및 두 동의 완료를 확정.
  • PASS 실패·중도 이탈 시: 동의·인증 모두 미완료로 유지, 예측 참여 차단(재시도 유도).
  • 본인인증이 이미 완료된 유저는 성별·생년월일이 확보돼 있으므로, 통계는 이용 목적 동의만 받으면 된다.
5수용기준
  • 미인증자가 예측 참여를 시도하면 본인인증+통계 동의 바텀시트가 노출된다.
  • 미인증·미동의자의 예측 상세 통계는 블러 상태이며, [확인하기] 또는 참여로 인증 완료 시 해제된다.
  • PASS 성공 전에는 예측 참여가 진행되지 않는다.
  • 통계 동의는 본인인증 동의와 분리된 항목으로 표시된다(목적 다른 동의 묶음 금지).
